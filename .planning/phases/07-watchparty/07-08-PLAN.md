<objective>
Close Issue #4 from 07-UAT-RESULTS.md — a product gap (not a bug): the watchparty participant strip and elapsed-time computation always anchor to `joinedAt`, but for pre-start or on-time joiners this is wrong (they joined before the party started; their real elapsed starts at `startAt`). For late joiners who were actually watching on TV already, there's no way to declare "I'm actually on time, I just hit Join late."

Two sub-features:
1. **Default on-time inference (passive, no Firestore write):** If a participant's `joinedAt` is ≤ `wp.startAt` (plus a 60s grace for realistic network latency), treat them as on-time: compute their displayed elapsed from `wp.startAt`, not `joinedAt`. Zero new state; pure render-time helper.
2. **Manual "I started on time" override (new UI + write):** If `joinedAt > wp.startAt + grace`, expose a one-tap control on the participant strip chip (or in the footer) that writes `participants.{uid}.effectiveStartAt = wp.startAt`. Elapsed now computes from that override. Idempotent — a "Never mind" / toggle-off reverts to null.

**Pattern citation (factual correction from checker):** `claimStartedOnTime` adopts a pre-await optimistic pattern — mutate local state + render BEFORE the await updateDoc. This matches 07-06's `setWpMode` fix and 07-07's `setReactionDelay`. It is DISTINCT from `postReaction`'s post-await echo at `js/app.js:7827-7832` (which awaits the write FIRST, then mutates `wp.reactions` and re-renders). Do NOT cite postReaction as the precedent — cite 07-06 / 07-07.

Purpose: The participant strip today shows "X min in" anchored to a ragged set of real joinedAt values — the family sees asymmetric timers even when they all "started the movie together." Issue #4 notes this is defensible but inferior. Fix makes the shared-anchor case the default.
Output: ~3 surfaces: new `effectiveStartFor(mine, wp)` helper + `computeElapsed` signature update (cascading to all 7 call sites) + `claimStartedOnTime` persist fn + UI control on the participant strip chip for late joiners.

const ONTIME_GRACE_MS = 60 * 1000;  // 60-second grace for network latency in on-time inference

function effectiveStartFor(participant, wp) {
  // Explicit override wins
  if (participant && typeof participant.effectiveStartAt === 'number') return participant.effectiveStartAt;
  // Default inference: joined at or shortly after startAt → treat as on-time, anchor to startAt
  if (participant && typeof participant.joinedAt === 'number' && typeof wp.startAt === 'number'
      && participant.joinedAt <= wp.startAt + ONTIME_GRACE_MS) {

}
