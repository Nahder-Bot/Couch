### 2. `dvrReadoutText()` — hour-aware extension

**Analog:** `js/app.js:10306-10310` (self-extension)

**Current function** (lines 10306-10310):
```javascript
function dvrReadoutText(sec) {
  if (!sec || sec <= 0) return 'No wait';
  if (sec <= 60) return `${sec} sec`;
  return `${Math.floor(sec/60)} min ${sec%60} sec`;
}
```

**Changes required:**
- Return `'Live'` (not `'No wait'`) when `sec <= 0` — per UI-SPEC copywriting contract
- Add hour branch: if `sec >= 3600`, compute `h = Math.floor(sec/3600)`, `m = Math.floor((sec%3600)/60)` → return `m ? \`${h} hr ${m} min\`` : `\`${h} hr\``
- Keep `sec < 60` branch returning `\`${sec} sec\``
- Keep `sec < 3600` branch returning `m ? \`${m} min ${s} sec\`` : `\`${m} min\`` (note: existing format adds trailing "sec" even when s=0 — clean that up)

### 5. `setReactionDelay()` — clamp widening

**Analog:** `js/app.js:11179-11194`

**Current function** (lines 11179-11194):
```javascript
window.setReactionDelay = async function(seconds) {
  const wp = state.watchparties.find(x => x.id === state.activeWatchpartyId);
  if (!wp || !state.me) return;
  const s = Math.max(0, Math.min(60, parseInt(seconds, 10) || 0));
  // Pre-await optimistic mutation
  if (wp.participants && wp.participants[state.me.id]) {
    wp.participants[state.me.id].reactionDelay = s;
  }
  renderWatchpartyLive();
  try {
    await updateDoc(watchpartyRef(wp.id), {
      [`participants.${state.me.id}.reactionDelay`]: s,
      lastActivityAt: Date.now(),
      ...writeAttribution()
    });
  } catch(e) { alert('Could not change delay: ' + e.message); }
};
```

**Single required change:** Line 11182 — change `Math.min(60, ...)` to `Math.min(86400, ...)`.

This is the **highest-priority code fix** in the phase. Without it, all UI inputs (expanded slider, new chips, picker, nudge) write values that Firestore silently caps at 60 sec.

### 14. `onWatchpartyUpdate` — reaction-push branch (cross-repo)

**Pattern — add Branch 2 BEFORE the `return null` guard:**
```javascript
// In onWatchpartyUpdate — add BEFORE the "return null" guard at line 225:
// Branch 2: New reaction appended to reactions array
const beforeLen = (before.reactions || []).length;
const afterLen  = (after.reactions || []).length;
if (afterLen > beforeLen) {
  const newReactions = after.reactions.slice(beforeLen);  // handles burst-append (R-6 mitigation)
  const participants = after.participants || {};
  const titleName = after.titleName || 'your show';
  for (const newReaction of newReactions) {
    if (!newReaction) continue;
    const senderId = newReaction.memberId;
    const senderName = newReaction.memberName || 'Someone';
    const reactionBody = newReaction.kind === 'emoji'
      ? newReaction.emoji
      : (newReaction.text || '');
    const tag = `reaction-${wpId}-${newReaction.id || Date.now()}`;
    const url = `/?wp=${wpId}`;
    // Split receivers by reactionDelay — must call sendToMembers TWICE (different bodies)
    const waitUpReceivers = Object.keys(participants)
      .filter(id => id !== senderId && (participants[id].reactionDelay || 0) > 0);
    const liveReceivers = Object.keys(participants)
      .filter(id => id !== senderId && !(participants[id].reactionDelay || 0));
    if (waitUpReceivers.length) {
      await sendToMembers(familyCode, waitUpReceivers, {
        title: senderName,
        body: `${senderName} is reacting to ${titleName}`,
        tag, url
      }, { excludeMemberId: senderId, eventType: 'reactionPosted' });
    }
    if (liveReceivers.length) {
      await sendToMembers(familyCode, liveReceivers, {
        title: senderName,
        body: `${senderName}: ${reactionBody}`,
        tag, url
      }, { excludeMemberId: senderId, eventType: 'reactionPosted' });
    }
  }
  return null;  // reaction branch handled — exit before the status branch
}
// Branch 1 (existing): status transition to 'active'
if (before.status === 'active' || after.status !== 'active') return null;
// ... existing code continues ...
```

**Critical:** The Branch 2 `return null` must be placed BEFORE the Branch 1 `return null` guard. This way reaction updates exit early rather than being silently dropped by the status guard.
