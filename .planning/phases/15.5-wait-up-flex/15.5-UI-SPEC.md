# Phase 15.5 — UI Design Contract

> Visual and interaction contract for the Wait Up flex UX (slider + chip strip + custom picker + nudge + Live reset + Past parties surface). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Voice anchors already in production (do not re-name):** slider label `Wait up` · chip-row label `Wait up` · catchup eyebrow `WAITED UP` · catchup dismiss `Got it. Catch me up.` · paused placeholder `Paused. We waited up.`

## Design System

| Property | Value |
|----------|-------|
| Tool | none (single-file no-bundler PWA — no shadcn, no npm deps) |
| Preset | not applicable |
| Component library | bespoke — Couch warm-dark design system (`BRAND.md` v1) |
| Icon library | none (text-only chips + glyph-free buttons; sole iconography is the `›` chevron in the Past parties link) |
| Font | `--font-sans` Inter (UI labels, chips, numeric inputs, buttons), `--font-display` Fraunces (Past parties surface title), `--font-serif` Instrument Serif (italic readouts only — kept lean per BRAND §3 anti-pattern) |

## Spacing Scale

Declared values (`BRAND.md` §4 semantic tokens — multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-inline-xs` | 4px | gap between numeric input fields' colon separators |
| `--space-inline-sm` | 8px | gap between chips inside the chip strip; gap between Live + first preset chip |
| `--space-inline-md` | 12px | gap between chip strip and `+5 min` nudge button; bottom-sheet horizontal inner padding floor |
| `--space-inline-lg` | 16px | bottom-sheet content horizontal padding (left + right of inputs); Past parties row inner padding |
| `--space-stack-sm` | 12px | vertical gap between the `Wait up` label and the chip row / slider readout |
| `--space-stack-md` | 20px | bottom-sheet section spacing (numeric-input row → preset row → action row); Past parties row vertical rhythm |
| `--space-stack-lg` | 28px | gap between Past parties surface header and the list itself |
| `--space-section` | 40px | gap between the active wp banner section and the Past parties inline link below it on Tonight tab |

Exceptions:
- **Touch target floor:** all chips (Live · presets · Custom… · `+5 min` nudge) MUST measure ≥44px tall (iOS HIG floor). The existing `.wp-control-btn.wp-delay` rule sets `min-width:44px;padding:4px 10px` (`css/app.css:841`) — Phase 15.5 keeps min-width at 44px AND raises min-height to 44px for tap reliability. This is an intentional override of the `--t-meta` 13px chip text size which would otherwise produce ~30px tall chips.
- **Bottom-sheet bottom inset:** sheet bottom padding = `max(var(--space-stack-md), env(safe-area-inset-bottom))` to clear the iOS home indicator on standalone PWA.

## Typography

Per `BRAND.md` §3 modular scale. Phase 15.5 uses 4 sizes + 3 weights (within the 4-sizes / 2-3-weights ceiling).

| Role | Token | Size | Weight | Line height | Usage in this phase |
|------|-------|------|--------|-------------|---------------------|
| Body | `--t-body` | 15px | 400 (regular) | 1.5 | numeric input field values; bottom-sheet body copy; Past parties wp-card title |
| Label / chip / button | `--t-meta` | 13px | 500 (medium) for off-state, 700 (bold) for on-state | 1.4 | chip labels (Live · 15 sec · 1 min · 5 min · 15 min · 30 min · 1 hr · Custom…); `+5 min` nudge button; sheet primary action; sheet cancel link |
| Eyebrow / readout | `--t-eyebrow` | 11px | 500, `text-transform: uppercase`, `letter-spacing: 0.08em` | 1.3 | `Wait up` row label; Past parties section eyebrow on Tonight tab |
| Heading (Past parties surface) | `--t-h2` | 22px | 400 (Fraunces is a display serif — its 400 reads heavy) | 1.2 | Past parties surface title `Past parties` |

Italic serif (`--font-serif` Instrument Serif) is reserved for ONE moment in this phase: the readout text when `reactionDelay > 0` reads in italic to carry the friend-voice register (e.g., *"holding 15 min"*). Off-state (Live) reads in regular Inter to stay quiet — restraint principle.

**Anti-pattern guard:** Do NOT mix Fraunces and Instrument Serif in the same visual section (BRAND §3). The Past parties surface uses Fraunces for the title only; everything below it stays Inter.

## Color

60/30/10 split per `BRAND.md` §2 warm-dark palette. Phase 15.5 introduces **zero** new color tokens — every color used is an existing semantic alias.

| Role | Token | Value | Usage in this phase |
|------|-------|-------|---------------------|
| Dominant (60%) | `--bg` / `--surface` / `--surface-2` | `#14110f` / `#1c1814` / `#25201a` | bottom-sheet body surface; Past parties list surface; chip-row container background; modal backdrop tint |
| Secondary (30%) | `--ink` / `--ink-warm` / `--ink-dim` | `#f5ede0` / `#c9bca8` / `#847868` | chip text (off-state `--ink-warm`); `Wait up` label (`--ink-dim`); numeric input text (`--ink`); Past parties metadata (`--ink-dim` for age/participant count) |
| Accent (10%) | `--accent` + `--brand-grad` | `#e8a04a` + brand gradient | **Reserved for the active states only** — see "Accent reserved for" list below |
| Destructive | `--bad` | `#c44536` | NOT USED in this phase. No destructive actions exist (cancel is non-destructive; setting Wait Up to 0 / Live is reset, not destruction). |

**Accent reserved for** (explicit list — never beyond):
1. The currently-selected chip's background (existing `.wp-control-btn.on` recipe at `css/app.css:836` uses `--brand-grad` — Phase 15.5 inherits this; only ONE chip is on-state at a time)
2. The slider's filled track + thumb when `reactionDelay > 0` (existing pattern — keep)
3. The bottom-sheet primary action button "Set wait" (`--brand-grad`, the ONE warm-shadow CTA per modal — `box-shadow: var(--shadow-warm)`)
4. The Past parties inline link's count badge — e.g., the `(3)` in `Past parties (3) ›` reads in `--accent` to draw the eye, while the rest of the link reads in `--ink-warm`

**Accent NOT used for:**
- The `+5 min` nudge button (uses surface `--surface-2` background + `--ink` text; brand-restraint says one accent per row)
- The Live chip (treated as the off-state baseline — quiet, in `--surface` + `--ink-warm`)
- The Custom… chip in its idle state (uses surface `--surface-2` + `--ink` text + dashed `--ink-faint` border to read as "input affordance" rather than "selected value")
- Past parties wp-card poster borders or row backgrounds

## Copywriting Contract

All copy follows `BRAND.md` §6: friend-on-couch register, sentence-case buttons, no em-dashes, no exclamation marks, no tech-mechanism words (`delay` / `buffer` / `queue` / `offset` / `sync` are BANNED in user-facing surfaces).

### Chip strip (movie mode) — final 8-element set

In order, left-to-right, wrap-flex on narrow viewports:

`Live` · `15 sec` · `1 min` · `5 min` · `15 min` · `30 min` · `1 hr` · `Custom…`

Notes:
- `Live` (capital L, no italic, no badge) — first chip, always-rendered, on-state when `reactionDelay === 0`
- `Custom…` uses a single-character horizontal ellipsis (U+2026), NOT three dots. This is the established convention in BRAND voice (sentence-case + restrained punctuation).
- Numeric chips use a thin space (or regular space) between number and unit (`15 sec` not `15sec`). Unit abbreviations: `sec`, `min`, `hr`. No plurals (`1 hr` not `1 hour`; `5 min` not `5 mins`).

### Chip strip (sport mode) — slider only

Sport mode keeps the existing slider scaffold + label `Wait up` (already in production per Wait Up rebrand commit `25df50c`). No chip strip; the slider IS the input. Step-ramp bands per `15.5-CONTEXT.md` D-01.

## Layout + Interaction Contracts

### 1. Sport-mode slider (extend existing `renderDvrSlider()` at `js/app.js:10290`)

- **Visual range:** Linear from 0 to 100 (slider's 0-100 position). Underlying transform maps position → seconds via the multi-segment step ramp from CONTEXT D-01.
- **Track:** `--surface-3` background (8px tall, `--r-pill` rounded). Filled portion `--brand-grad` from 0 to thumb position. Empty portion `--ink-faint` at 0.4 alpha.
- **Thumb:** 24px circle, `--accent` background, `--shadow-soft`. Active drag scales to 1.1x at `--duration-fast` ease-out.
- **Layout:** Above slider — `Wait up` eyebrow row. Below slider — readout (italic *holding {value}* when active, regular `Live` when 0) + inline `+5 min` button to the right of the readout (visible only when `reactionDelay > 0`).
- **Vertical rhythm:** `--space-stack-sm` between eyebrow → slider → readout. `--space-inline-md` between readout and `+5 min` button.

### 2. Movie-mode chip strip (extend existing chip pattern at `js/app.js:11023-11045` + CSS at `css/app.css:839-841`)

- **Container:** `.wp-delay-row` already exists. Phase 15.5 keeps `display:flex; gap:8px; flex-wrap:wrap; align-items:center;` (raise gap from 6px to 8px to match `--space-inline-sm`).
- **Chip count:** 8 elements (Live · 15 sec · 1 min · 5 min · 15 min · 30 min · 1 hr · Custom…). Wraps to 2-3 rows on iPhone-mini-class viewports (~320-375px wide). Wrap is intentional — the strip is allowed to be tall on narrow phones; it stays one row on iPhone-Plus-class and wider.
- **Chip dimensions:** ≥44px tall (touch target floor), `min-width: 44px`, `padding: 8px 12px`, `--r-pill` radius, `--t-meta` text size.
- **Off-state:** `--surface-2` background, `--ink-warm` text, 1px `--ink-faint` border at 0.5 alpha. Weight 500.
- **On-state:** `--brand-grad` background, `--bg` text, no border, weight 700, optional inset `0 1px 0 rgba(255,255,255,0.15)` highlight (matches existing `.wp-control-btn.on` recipe at `css/app.css:836`).
- **Custom… chip idle:** `--surface-2` background, `--ink` text, 1px `--ink-faint` DASHED border (signals "input field" affordance vs solid-border preset chips). When the user has set a custom value via the picker, Custom… exits idle state and reads as the ON state of whatever value was set; it never simultaneously shows the value AND remains in idle dashed state.
- **Sub-line below chip strip:** italic *waiting {value}* readout when `reactionDelay > 0`, plus inline `+5 min` button to the right. Hidden when `reactionDelay === 0`. `--space-stack-sm` from chip row.
- **Tap behavior:**
  - Tap any preset chip → write value to `participants[me].reactionDelay` (throttled per existing `setDvrOffset` pattern), update on-state styling, re-render readout.
  - Tap `Live` → write 0, update readout to `Live`, hide `+5 min` button + sub-line.
  - Tap `Custom…` → open bottom-sheet picker (slide-up). Existing chip remains visible behind the backdrop.
- **Wrap rendering:** When chips wrap to row 2+, the row-2+ left edge aligns with the row-1 left edge (no centered wrap). Row gap = `--space-inline-sm` (8px).

### 3. Bottom-sheet custom picker (NEW — opens on Custom… chip tap)

- **Pattern:** Reuse existing `.modal` / `.modal-bg` scaffold per `BRAND.md` §7. Sub-class as `.wait-up-picker-modal` (or similar) for sheet-specific positioning.
- **Sheet position:** Bottom-anchored. `width: 100%`, `max-width: 480px` (centered on tablet/desktop). `border-radius: var(--r-md) var(--r-md) 0 0` (top corners rounded only). Bottom edge flush.
- **Sheet height:** `min-height: 320px`, `max-height: 70vh`, `height: auto` based on content.
- **Sheet inner padding:** `var(--space-stack-md) var(--space-inline-lg)` (vertical 20px, horizontal 16px). Bottom: `max(var(--space-stack-md), env(safe-area-inset-bottom))` to clear iOS home indicator.
- **Backdrop:** `.modal-bg` darkens the rest of the screen at 0.6 alpha. Tap dismisses without commit.
- **Backdrop fade:** `--duration-base` (220ms) ease-out. Sheet slide: `--duration-deliberate` (300ms) ease-out per BRAND §5.
- **Internal layout (top to bottom):**
  1. **Header row** (`--space-stack-sm` bottom margin): Fraunces 22px title `How long should we wait up?` aligned left.
  2. **Numeric input row** (`--space-stack-md` vertical padding inside): three `<input type="text" inputmode="numeric" pattern="[0-9]*">` fields side-by-side. Each field 64px wide × 56px tall, `--r-sm` radius, `--surface-3` background, `--ink` text 24px Inter regular center-aligned. Below each field: tiny `--t-eyebrow` label `HOURS` / `MIN` / `SEC` in `--ink-dim`. Gaps between field groups = `--space-inline-md`.
  3. **Helper line** (italic Instrument Serif, `--ink-dim`, `--t-meta`): *Only changes when you see reactions. Never anyone else's view.*
  4. **Eyebrow** (`--t-eyebrow`, `--ink-dim`, all-caps): `OR JUMP TO`.
  5. **Preset row** (chip set, same chip recipe as main strip): `1 hr` · `2 hr` · `12 hr` · `24 hr`. Tap any preset → fills all three numeric fields with the corresponding values AND immediately submits (one-tap commit; closes sheet).
  6. **Action row** (sticky to sheet bottom, `--space-stack-md` top padding): `Cancel` link on the LEFT (`--ink-dim`, no underline, sentence case), `Set wait` primary button on the RIGHT (`--brand-grad`, `--shadow-warm`, `--r-md`, padding `12px 24px`, weight 700). Min 44px tap height.
- **Submit behavior:** "Set wait" computes total seconds = `hours * 3600 + minutes * 60 + seconds`. Clamps to `[0, 86400]`. Writes to `participants[me].reactionDelay` (throttled). Closes sheet. Updates main chip strip to show new state (Custom… exits idle dashed style and reads as on-state with the new value visible in the sub-line readout).
- **Keyboard input:** `inputmode="numeric"` + `pattern="[0-9]*"` triggers iOS Safari + Android Chrome native number keypad. Tab order: hours → min → sec → Set wait. Submit-via-Enter from any field commits.
- **Validation:** Each field accepts 0-99 (allow > 24 in hours field; clamp on submit, not on input — input feels lossless). Empty field treated as 0. Negative / non-numeric input rejected via `pattern`.
- **Dismiss without commit:** backdrop tap, Cancel link tap, Android back gesture, or browser back. Existing `reactionDelay` value preserved.

### 4. `+5 min` nudge button

- **Visibility:** Renders ONLY when `reactionDelay > 0`. Hides immediately when value reaches 0 (Live) or when chip strip / slider is reset.
- **Position:**
  - Sport mode: inline right of the readout text below the slider.
  - Movie mode: inline right of the readout text in the sub-line below the chip strip.
- **Style:** `--surface-2` background (NOT accent — see "Accent NOT used for" above), `--ink` text, 1px `--ink-faint` border, `--r-pill` radius, `padding: 8px 12px`, ≥44px tall touch target. Weight 500 Inter.
- **Tap behavior:** Adds 300 sec to current `reactionDelay`. Clamps at `86400` (24 hr). Throttled write via existing setter pattern. Readout updates immediately; if value crosses chip-preset boundary (e.g., from 1 min to 6 min), the on-state chip styling shifts to whichever preset matches OR all chips go to off-state if value lands between presets (Custom-style).
- **Haptic:** Light tap on success per existing `haptic()` util in `js/utils.js`. None on clamp (no haptic for "did nothing").
- **Tap feedback:** `--duration-fast` (150ms) scale to 0.96 then back per existing button-press recipe.

### 5. Live reset semantic

- Live is the FIRST chip in the movie-mode strip (always position 0).
- In sport mode, Live is the slider-at-0 state (no separate chip needed; slider thumb at the leftmost position IS the Live state, readout reads `Live`).
- Tap Live in movie mode → write 0 to `reactionDelay`, hide `+5 min` button, hide sub-line readout.
- Live chip on-state styling = same on-state recipe as other chips (`--brand-grad` background) BUT only when `reactionDelay === 0`. The default state of the strip on a fresh wp join is Live being the ON chip.

### 6. Past parties surface

#### 6a. Tonight tab inline entry point

- **Position:** Inserted immediately below the active wp banner section, separated by `--space-section` (40px) from the active wp banner above and the next Tonight tab section below.
- **Render condition:** Only render when 1+ watchparties exist where `5h ≤ (now - startAt) < 25h`.
- **Layout:** Single-line link, left-aligned. Format: `Past parties (N) ›` where N is the count.
- **Style:** Inter 15px regular, `--ink-warm` for text, `--accent` for the `(N)` count, `--ink-dim` for the `›` chevron. No background, no border, no underline. Tap target: 44px tall full-width row with implicit padding `12px 16px`.
- **Tap behavior:** Routes to the dedicated Past parties surface (new screen). Browser back / iOS swipe-back returns to Tonight tab.

#### 6b. Past parties surface (NEW screen)

- **Pattern tier:** Tab-section per BRAND §7 tier 2 (NOT a modal; this is a navigated screen). Reuses `.tab-hero` / `.tab-hero-title` / `.tab-section` scaffold.
- **Header:**
  - Eyebrow: `STILL ROLLING` (`--t-eyebrow`, `--ink-dim`, all-caps via CSS).
  - Title: `Past parties` (Fraunces `--t-h2` 22px, `--ink`).
  - Back affordance: existing left-chevron back button per app shell convention. Aria-label `Back to Tonight`.
- **List body:** Reuses existing wp-card / `.tab-list-card` patterns. Each row:
  - Poster thumb: 56px × 84px (3:2 aspect), `--r-sm` radius, left-aligned.
  - Right-side stack (vertical, `--space-stack-sm` rhythm):
    - Title (Inter 15px regular `--ink`)
    - Subtitle line 1: `Started {N} hr ago` (Inter 13px `--ink-dim`)
    - Subtitle line 2: `{N} on the couch` (Inter 13px `--ink-dim`)
  - Right edge: `›` chevron in `--ink-faint`, vertically centered.
  - Row padding: `var(--space-stack-sm) var(--space-inline-lg)` (12px / 16px).
  - Row separator: 1px solid `--ink-faint` at 0.3 alpha bottom border. Last row no border.
  - Min row height: 96px (poster height + vertical padding) — comfortably exceeds 44px touch floor.
- **Tap behavior:** Opens the wp the same way the active-banner tap does today. Wp itself renders normally (no Past-parties-specific wp UI; entering a stale wp is the same experience as entering an active wp).
- **Empty state:** Surface is unreachable when zero stale wps exist (the inline link doesn't render). Documented as intentional zero-clutter pattern. No empty-state copy needed.
- **Sort order:** Most recent first (largest `startAt`, smallest `now - startAt`). The wp that just crossed 5h sits at the top.

## Accessibility Contract

- **Touch target floor:** ≥44pt (iOS HIG) for every interactive element introduced. Includes chips, nudge button, picker numeric inputs, picker preset chips, picker action buttons, Past parties inline link row, Past parties wp-card row.
- **Aria-labels:** Every chip, slider, picker input, nudge button, and link carries a `aria-label` per the table in Copywriting Contract above.
- **Focus order:** chip strip flows left-to-right, top-to-bottom (matches visual wrap order). Bottom-sheet focus order: title → hours → min → sec → preset chips left-to-right → Cancel → Set wait. On open, focus lands on the hours field (mobile keypad pops). On close, focus returns to the Custom… chip.
- **Screen-reader announcements:**
  - Chip on-state change announces new value: `Wait up: 15 min, currently active`.
  - `+5 min` tap announces new total: `Wait up now 6 minutes`.
  - Live reset announces: `Wait up off, back to live`.
  - Picker submit announces: `Wait up set to 1 hour 30 minutes`.
- **No banned words in aria-labels:** Same banned-words ledger applies. The aria-label `Wait up by, current value X` uses a comma (not em-dash) per BRAND voice.
- **Color contrast:** All text/background pairs MUST be verified against `webaim.org/resources/contrastchecker/` per BRAND §2. Phase 15.5 introduces no new color combos; relies on existing semantic tokens which all pass AAA at the levels they're used for. Specific verifications:
  - Chip off-state text (`--ink-warm` on `--surface-2`): 8.5:1 ✓ AAA
  - Chip on-state text (`--bg` on `--brand-grad` midpoint `#d97757`): ≥5.5:1 ✓ AA (pass for 14px+ text)
  - Numeric input text (`--ink` on `--surface-3`): ≥12:1 ✓ AAA
  - Past parties subtitle (`--ink-dim` on `--bg`): ~4.0:1 ✓ AA Large; subtitle is 13px which is below "Large" threshold (18pt = 24px) — DOWNGRADE FAIL RISK. **Mitigation:** use `--ink-warm` (`#c9bca8`, 8.5:1) for subtitles, NOT `--ink-dim`. This UI-SPEC overrides the initial `--ink-dim` choice for subtitles to `--ink-warm`. Update the table in §6b accordingly when planner implements.
- **Reduced motion:** All animations clamp to 0.01ms under `prefers-reduced-motion: reduce`. Sheet appears instantly, no slide. Chip state changes are instant.
