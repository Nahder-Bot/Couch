## Decisions Made

- **`type="text"` + `inputmode="numeric"` (NOT `type="number"`):** UI-SPEC + RESEARCH §REQ-3 + RESEARCH §R-2 all converged on this — iOS Safari shows a decimal-with-minus keypad for `type="number"` which is wrong for hours/min/sec entry. The text+inputmode combo gives the integer-only keypad on iOS + Android. `pattern="[0-9]*"` is the documented attribute for the iOS numeric-keyboard hint.
- **`maxlength="2"` on each field:** Constrains hours/min/sec to 2 digits each (max raw value 99 each). Hours field at "99" maps to 99×3600=356400 sec; the submit clamp catches the > 24h overflow safely. Per UI-SPEC § Validation: "allow > 24 in hours field; clamp on submit, not on input — input feels lossless."
- **60ms setTimeout before focus + scrollIntoView:** Without the delay, focus fires before the `.modal-bg.on` class CSS has applied + the sheet has started its slide-up animation. Result: keyboard pops mid-slide, visual jitter. The 60ms is enough to let the first animation frame paint; longer delays would feel laggy. Tested mentally against Plan 02's `--duration-base` 220ms backdrop fade — 60ms is well under that, so the keyboard appears while the sheet is still settling, which is the desired feel.
- **Cancel button does NOT carry `.modal-close` class:** Only the Set wait (commit) button uses `.modal-close` to inherit `--brand-grad` + `--shadow-warm`. Cancel uses `.wup-cancel` link styling (no background, `--ink-dim`, no underline) per UI-SPEC § Action row. One accent CTA per modal — the BRAND restraint principle.
- **Defensive try/catch on focus + scrollIntoView:** Both are best-effort UX polish. If they throw (very unlikely — element exists, both methods are universally supported), the picker still opens with a populated input row + visible helper line + tappable presets and Set wait. The catch comments mark them as defensive.
- **Empty fields render as empty strings (not `'0'`):** `if (hEl) hEl.value = h ? String(h) : '';` shows the placeholder `0` when the value is 0, instead of overwriting with a literal `0`. Less visual noise; user types over an empty field rather than selecting-and-replacing a literal zero.
- **Defense-in-depth clamping at three layers:** `setPickerPreset` clamps input `sec`, `submitWaitUpPicker` clamps the computed `h*3600+m*60+s` total, AND `setReactionDelay` (Plan 01) clamps once more at the Firestore-write boundary. Belt-and-suspenders. Per the threat model's T-15.5-01 mitigation: even if `maxlength="2"` is bypassed via DevTools (e.g. `document.getElementById('wup-hours').value = '999999999'`), the parseInt + clamp catches it.
- **Reduced-motion override cancels sheet + backdrop animation only:** Chip taps inside the picker still rely on Plan 03's existing `.wp-control-btn.wp-delay` reduced-motion handling. No new override needed for chip transitions.
