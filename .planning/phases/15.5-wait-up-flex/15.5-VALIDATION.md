# Phase 15.5 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Couch has no automated test suite (single-file no-bundler PWA). All validation is manual UAT (mobile devices) or Cloud Functions emulator runs from the sibling `queuenight/` repo. Per-task table is populated by the planner after PLAN.md files exist.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | None (manual UAT + Firebase Functions emulator) |
| **Config file** | None on couch side; `~/queuenight/firebase.json` for emulator |
| **Quick run command** | Manual: open `couchtonight.app/app` on test device after `bash scripts/deploy.sh <tag>` |
| **Full suite command** | Two-device UAT session — see Scripts A–E in `15.5-RESEARCH.md` § Validation Architecture |
| **CF emulator command** | `cd ~/queuenight && firebase emulators:start --only functions` |
| **Estimated runtime** | Manual UAT: ~25 min for full Scripts A–E pass on two devices |

---

## Sampling Rate

- **After every task commit:** No automated test → run `node -c <modified.js>` syntax check; for CSS, deploy preview channel (`firebase hosting:channel:deploy`) and visually confirm.
- **After every plan wave:** Full UAT script for any user-visible behavior touched in the wave (Scripts A–E in `15.5-RESEARCH.md`).
- **Before `/gsd-verify-work`:** All five UAT scripts pass on iPhone-mini-class device (375px) AND a tablet/desktop viewport. Push-stripping (Script D) verified on two physical devices OR the CF emulator with mocked receiver `reactionDelay`.
- **Max feedback latency:** ~30 seconds (deploy preview channel build + cache-bust open).
