<objective>
Plan 05 — Push template content stripping (REQ-7 + REQ-8). Cross-repo: modifies `queuenight/functions/index.js` (sibling repo) AND `js/app.js` for the client-side notification-prefs mirror (DR-3 pattern from Phase 14).

**The architectural gap RESEARCH.md flagged:** there is NO existing reaction push notification today. Reactions are appended to `wp.reactions` via `arrayUnion` (js/app.js:11115); there is no reaction sub-collection. So this plan BUILDS the reaction-push channel from scratch by adding a new `Branch 2` to the existing `onWatchpartyUpdate` Cloud Function — detecting when `after.reactions.length > before.reactions.length` and fanning out a push for each new reaction.

The stripping happens by calling `sendToMembers` TWICE per reaction:
- Once for receivers where `participants[id].reactionDelay > 0` → body `"{SenderName} is reacting to {TitleName}"` (no spoiler content)
- Once for receivers where `reactionDelay === 0` → body `"{SenderName}: {emoji|text}"` (full content)

The sender is excluded from both (self-echo guard).

Plan 05 is **non-autonomous** — Cloud Functions cannot deploy without user-side `firebase deploy --only functions` from `~/queuenight/`. The deploy ritual is part of Plan 07 (`commit + cross-repo deploy`); Plan 05's own work is the code change in both repos.

**Cross-repo deploy ordering (R-8):** queuenight functions MUST deploy BEFORE couch hosting. If the order reverses, clients with `reactionDelay > 0` will write large values that the OLD CF (without strip-branch) doesn't handle — early users get spoiler-content pushes during the deploy gap. Plan 07 enforces this ordering in the deploy ritual.

Purpose: Close the spoiler-on-lock-screen gap so that when a receiver has Wait Up active, push notifications stop landing the reaction body N minutes/hours before they're supposed to see it in-app.

Output: `queuenight/functions/index.js` modified (NOTIFICATION_DEFAULTS + onWatchpartyUpdate Branch 2). `js/app.js` modified (DEFAULT_NOTIFICATION_PREFS + NOTIFICATION_EVENT_LABELS DR-3 mirror).

        if (afterReactions.length > beforeReactions.length) {

            const reactionId = newReaction.id || `${Date.now()}`;
            const tag = `reaction-${wpId}-${reactionId}`;
            const url = `/?wp=${wpId}`;
            // Split receivers into two groups by reactionDelay.
            const allReceivers = Object.keys(participants).filter(id => id !== senderId);
            const waitUpReceivers = allReceivers.filter(id => (participants[id] && participants[id].reactionDelay) > 0);

            // Strip-body fan-out: receivers with Wait Up active. No reaction content; sender-attributed.
