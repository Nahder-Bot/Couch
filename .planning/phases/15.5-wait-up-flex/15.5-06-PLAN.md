<objective>
Plan 06 — Stale-wp 5h cutoff filter on Tonight tab + Past parties surface (D-04 + REQ-9 + REQ-10).

Today's Tonight tab `renderWatchpartyBanner()` (js/app.js:10543-10611) shows ALL active watchparties up to the 25h `WP_ARCHIVE_MS` window. With Wait Up extending offsets to 24h, multiple parties can stack and clutter the surface. Plan 06 splits the rendered set:
- **Fresh wps** (banner): `wp.startAt > now` OR `(now - wp.startAt) < 5h` OR `wp.status === 'cancelled'` (preserves existing cancelled-wp 10-min sticky window per the existing render logic)
- **Stale wps** (Past parties surface): `wp.status !== 'cancelled'` AND `(now - wp.startAt) >= 5h` AND `(now - wp.startAt) < 25h`

When stale wps exist, an inline link `Past parties (N) ›` renders below the banner section. Tap opens a new modal surface listing the stale wps with poster + title + age + participant count.

**Routing approach (per RESEARCH § REQ-9 recommendation):** Past parties is a tall `.modal-bg` sheet, NOT a `showScreen` destination. Avoids un-highlighted tab-bar state issue and gives free hardware-back / iOS-swipe-back dismissal. UI-SPEC § 6b describes a "Tab-section per BRAND §7 tier 2" — the modal implementation achieves the same visual result (tall full-width sheet, header + list) without the routing complications.

`activeWatchparties()` itself is NOT modified — the filter is applied at render time inside `renderWatchpartyBanner` and inside `renderPastParties`. This preserves the canonical 25h archive boundary.

Edge case (Claude's discretion per CONTEXT): a user actively inside a wp's live modal when it crosses the 5h boundary keeps that modal open — only the BANNER state recategorizes. On modal close, the wp shows up in Past parties rather than the main banner. Per RESEARCH § REQ-9: "the live modal doesn't observe banner changes" — no special handling needed.

Plan 06 is autonomous. No dependency on Plans 01-05 (the stale-wp logic is independent of the Wait Up input UI). However, Plan 06 lives in Wave 2 because the Tonight tab is user-visible and ships best alongside the rest of the user-visible work.

Purpose: Keep Tonight tab uncluttered as Wait Up extends watchparty long-tail; surface long-tail wps in their own discoverable surface.

    function renderPastParties() {
      const el = document.getElementById('past-parties-list');
      if (!el) return;
      const now = Date.now();
      // Use the same WP_STALE_MS boundary as renderWatchpartyBanner — single source of truth.
      const stale = activeWatchparties().filter(wp =>

        wp.startAt <= now &&
        (now - wp.startAt) >= WP_STALE_MS
