## addWaitUpNudge Clamp Behavior

```
current = mine.reactionDelay  // [0, 86400]
if (current <= 0) return       // defensive guard (button is hidden anyway)
newVal = Math.max(0, Math.min(86400, current + 300))
wasClamped = (current + 300) > 86400
haptic only if (!wasClamped)   // UI-SPEC: no haptic for "did nothing"
```

Examples:
- `current=0` → returns early (button hidden) — 0 writes
- `current=300` (5 min) → `newVal=600` (10 min), haptic fires
- `current=85800` (23h50m) → `newVal=86100` (23h55m), haptic fires
- `current=86200` (23h56m40s) → `newVal=86400` (24h cap), `wasClamped=true`, no haptic
- `current=86400` (already at cap) → `newVal=86400`, `wasClamped=true`, no haptic

## Decisions Made

- **U+2026 ellipsis as JS string literal:** `Custom…` written as a single-char `…` in the source. No HTML entity, no three-dot fallback. Verified the source byte appears as `…` (UTF-8 0xE2 0x80 0xA6) in `js/app.js:11096` (in the comment) and at `js/app.js:11112` (in the chip onclick label).
- **Sub-line render co-located with chip row in the same template literal:** kept both the row HTML and the sub-line HTML inside `renderWatchpartyFooter`'s output template. Avoids a separate render-target DOM id, avoids a second `getElementById` write path. When `setReactionDelay` mutates the value and calls `renderWatchpartyLive()`, both row and sub-line re-render together.
- **Forward-dependency stub for `openWaitUpPicker`:** Plan 04 ships the function definition. Until Plan 04 deploys alongside Plan 03 in Wave 2, tapping Custom… is a JS no-op (function name is undefined). Coordinated by Wave 2 deploy ritual in Plan 07. No Sentry instrumentation added — same-wave coordination eliminates the production gap window.
- **Live chip dual-variant aria-label:** UI-SPEC element-level copy table specifies BOTH `'Wait up: Live, currently active'` (on-state, regular cell) AND `'Stop waiting up, go back to live'` (off-state-as-CTA, separate cell). Implemented via ternary: `s === 0 ? (on ? '...' : '...') : (on ? '...' : '...')`. Custom chip got the same treatment but with `'Wait up: open custom picker'` baseline.
- **Sub-line negative-top-margin (-6px):** Matches the parent `.wp-delay-row`'s `margin: -6px 0 12px` to keep row + sub-line as a visual unit. The standard 12px bottom margin still applies under the sub-line.
- **Defensive `if (current <= 0) return;` in `addWaitUpNudge`:** Even though the button is hidden when `reactionDelay === 0`, a state-tearing edge case (e.g., async write race between `setReactionDelay(0)` and `addWaitUpNudge` taps) could land the function with `current === 0`. The guard avoids writing `300` from a "supposedly Live" state. Tested mentally — does the right thing.
- **`qnLog` instead of `alert` for nudge failures:** `setReactionDelay` uses `alert()` because chip taps are explicit and a silent failure is confusing. Nudge taps are inferred (user wants to extend the wait); a failure is a non-critical UX glitch best logged silently to Sentry-via-qnLog. Matches `setDvrOffset` slider-write error handling.
