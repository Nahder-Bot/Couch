**Critical scope deltas vs the original (now-deleted) Plan 28-05:**

1. **DROP UFC picker variant entirely.** No `ufc_winner_method` selection card, no UFC
   fighter chip row, no UFC method chip row, no UFC roster fetch. Per D-17 update 2,
   UFC was dropped from Phase 28 v1 because no free UFC API exposes structured roster
   + settlement at zero cost. The Pick'em tab MUST filter the LEAGUES catalog to
   exclude `'ufc'` at render time:
   ```js
   const PICKEM_LEAGUES = leagueKeys.filter(k => k !== 'ufc');
   // Iterate PICKEM_LEAGUES instead of leagueKeys when rendering league sections.
   ```
   UFC itself stays in `js/sports-feed.js` LEAGUES catalog because the existing
   watchparty mode (Phase 22) uses it. Only the pick'em surface filters it out.

2. **F1 picker = Jolpica integration.** The `f1_podium` variant uses a 3-row dropdown
   stack (P1 / P2 / P3). Each dropdown's `<option>` list comes from a fresh fetch
   against `https://api.jolpi.ca/ergast/f1/{year}/{round}/drivers/` at picker render
   time. Returned drivers have shape `{ givenName, familyName, code, permanentNumber, nationality }`.
   Display label: `givenName + " " + familyName` (e.g. "Max Verstappen") — the SAME
   canonical form Plan 28-03's gameResultsTick uses for podium settlement, so
   string-equality holds across the round-trip (Pitfall 6).

3. **Failure-soft for Jolpica:** if the fetch fails (network error, 5xx, timeout), the
   F1 picker card shows the locked-empty state:
   ```
   "Race entry list unavailable; check back later."
   ```
   Other pickTypes are unaffected. Submit is disabled for that card; user can still
   pick on team-sport / soccer cards on the same page.

4. **REQ-IDs renumbered.** The original PICK-28-06 (UFC variant) is gone. PICK-28-27
   (UFC filter at render) is new. PICK-28-04 narrowed from "4 pickTypes" to "3
   pickTypes".

5. **F1 picks denormalize `f1Year` + `f1Round`** at submit time so Plan 28-03's
   `gameResultsTick` doesn't have to derive them from gameId. These two fields are
   member-writable at CREATE per Plan 28-04 rule (not in the CF-only denylist).

6. **Cache target irrelevant in this plan** — Plan 28-06 owns the sw.js bump to
   `couch-v47-pickem`. This plan does NOT touch sw.js.
