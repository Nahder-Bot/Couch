**Critical scope deltas vs the original (now-deleted) Plan 28-04 + REVIEWS amendments:**

- **REVIEWS Amendment 6 / HIGH-1** — drop the `(!('state' in request.resource.data) || ...)`
  omission branch. State must be explicitly `'pending'` on create. Same tightening for
  `pointsAwarded`, `settledAt`, `tiebreakerActual` — required-equals, not optional-or-equals.
  This closes the silent-leaderboard-pollution gap (state-omitted picks were invisible to
  gameResultsTick).
- **REVIEWS Amendment 7 / HIGH-9** — replace the affectedKeys DENYLIST with an ALLOWLIST.
  The previous rule allowed mutation of `pickType`, `gameId`, `leagueKey`, `strSeason`,
  `f1Year`, `f1Round`, `gameStartTime`, `submittedAt`. New rule allows ONLY
  `{selection, tiebreakerTotal, editedAt, actingUid, memberId, memberName}`. All other
  fields are immutable post-create. (Note `actingUid`/`memberId`/`memberName` are in the
  allowlist because the existing `attributedWrite` helper requires them on every write —
  but they MUST equal the previous values; rule enforces that separately via the
  `resource.data.memberId == request.resource.data.memberId` check.)
- **REVIEWS Amendment 8 / HIGH-4** — add `pickType in ['team_winner', 'team_winner_or_draw', 'f1_podium']`
  on create. UFC pickType create denied at the rules layer. Defense-in-depth alongside
  Plan 28-03's backend KNOWN_PICKTYPES skip.
- **REVIEWS Amendment 9 / MEDIUM-8** — expand rules-tests from 4 to 10 covering: (a)
  create-before-lock OK; (b) create-after-lock denied; (c) CF-only field tamper denied;
  (d) non-family read denied; **(e) state-omitted create denied;** **(f) UFC pickType
  create denied;** **(g) gameStartTime update denied;** **(h) pickType update denied;**
  (i) leaderboards write denied; (j) picks_reminders read denied. Total rules-tests count
  grows 78 → 88 (was 78 → 82 in original plan).
