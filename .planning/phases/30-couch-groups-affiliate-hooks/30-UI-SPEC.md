# Phase 30 — UI Design Contract — Couch Groups

> Visual and interaction contract for cross-family watchparties (couch groups). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope reminder (per CONTEXT D-01):** Phase 30 ships **couch groups only**. The original "+ affiliate hooks" half is carved out to a future Phase 30.1 and produces ZERO UI surface in this phase. The phase directory keeps its `30-couch-groups-affiliate-hooks` slug only because rename is deferred (D-03); the contract below covers groups exclusively.

Phase 30 uses Couch's locked type scale (BRAND § 3). **No new sizes, no new weights.**

| Role | Token | Size | Weight | Line Height | Phase 30 Usage |
|------|-------|------|--------|-------------|----------------|
| Body | `--t-body` | 15px | 400 (Inter) | 1.5 | Default UI text in family-add input + form labels |

## Color

Couch's locked palette (BRAND § 2) carries Phase 30 with one render-time addition: cross-family member avatars use the foreign family's stored member color (denormalized to `wp.crossFamilyMembers[i].color` per RESEARCH Pattern 2). No new color tokens introduced.

| Role | Token | Value | Phase 30 Usage |
|------|-------|-------|----------------|
| Dominant (60%) | `--bg` | `#14110f` | App background (unchanged) |
| Secondary (30%) | `--surface`, `--surface-2` | `#1c1814`, `#25201a` | Form surfaces, chip backgrounds (unchanged) |
| Accent (10%) | `--accent` | `#e8a04a` (amber) | **Reserved-for list below** |
| Brand-voice italic | `--ink-warm` | `#c9bca8` | Italic tagline + collision-suffix `(Smiths)` italic span |
| Muted meta | `--ink-dim` | `#847868` | Secondary chip text (existing); cross-family member's family-code label if shown anywhere |
| Destructive | `--bad` | `#c44536` | "Remove this couch" host action confirm button (Path B kick flow per CONTEXT § Claude's Discretion) |
| Good (success) | `--good` | `#7fb069` | "Couch added" success toast on `addFamilyToWp` CF success |
| Warn (soft cap) | `--warn` | `#d4a76a` | Soft-cap warning text "That's a big couch — are you sure?" at 5+ families (D-08) |

### Accent reserved for (Phase 30):

Phase 30 does **NOT** introduce a new accent surface. Existing accent reservations from BRAND § 2 carry forward unchanged. Specifically for Phase 30:

1. The "+ Add a family" button styled as a **secondary `.alt` action** (NOT brand-gradient) — this is a host-only utility action, not a brand hero moment per BRAND § 7 Tier 2. It uses `--surface-2` background with `--ink` text and `--border` outline. **No accent.**
2. Cross-family member chips use **the foreign family's stored avatar color** (denormalized per RESEARCH Q4) — NOT `--accent`. This visually distinguishes families without burning the accent budget.
3. `.wp-participant-chip.me` retains its existing `rgba(232,160,74,0.45)` accent border for the current viewer's own chip (unchanged from Phase 7).
4. **No new accent surface added.** Per BRAND § 8 ("DON'T: gradient tabbar + gradient CTA + gradient hero in one view"), the existing accent budget is already spent.

### Brand-voice tightrope (CONTEXT-derived):

CONTEXT § Specifics calls out the visual reinforcement choice for D-09 collision-suffix: **italic Instrument Serif at `--ink-dim` contrast** rather than a colored badge-pill. This contract locks that choice. No `(Smiths)` badge, no family-color tinted suffix. Pure type-treatment disambiguation.

## Copywriting Contract

All copy follows BRAND § 6 voice rules (sentence-case buttons, no exclamation marks except celebratory moments, italic Instrument Serif for brand-voice lines). Copy is locked at the contract level so the planner does not need to re-derive it.

### Primary surfaces

| Element | Copy | Voice Note |
|---------|------|------------|
| Section heading (wp-create lobby + wp-edit) | **Bring another couch in** | Fraunces Heading 17px. Carries the "two families watching together" mental model from CONTEXT D-04. |
| Sub-heading italic (under section heading) | *Pull up another family for tonight.* | Instrument Serif italic 17px at `--ink-warm`. Honors BRAND § 7 Tier-2 sub-line pattern. |
| "+ Add" button label | **+ Add a family** | Sentence-case per BRAND § 6. The `+` glyph is part of the button label string. |
| Family-code input placeholder | **Paste their family code** | Echoes the existing Phase 5 family-join screen language (CONTEXT § Carrying forward). |
| Inline help under input | *They'll see their crew show up in this watchparty's couch.* | Italic Instrument Serif at `--ink-dim`, 13px. Sets the "what happens next" mental model. |
| Submit button (inside form) | **Bring them in** | Sentence-case warm verb. Avoids transactional "Add family." |

### Per-family row (in the "Couches in this party" list)

| Element | Copy | Voice Note |
|---------|------|------------|
| Row label (own family) | **{FamilyName} (your couch)** | Plain Inter; "(your couch)" suffix in `--ink-dim` to mirror the existing "(you)" pattern at `js/app.js:12339`. |
| Row label (foreign family) | **{FamilyName}** | Plain Inter at `--ink`. No badge. |
| Host kebab → "Remove" menu item | **Remove this couch** | Sentence-case. NOT "Kick" or "Eject" — both feel hostile and break the "friends + family" frame. |
| Remove confirmation modal heading | **Remove the {FamilyName} couch?** | Question form softens the destructive moment. |
| Remove confirmation modal body | *Their crew will lose access to this watchparty. They can be added back the same way.* | Italic Instrument Serif sub-line — sets the reversibility expectation. |
| Confirm button | **Yes, remove them** | Sentence-case. Pairs with `--bad` color for the destructive class. |
| Cancel button | **Keep them** | Warm cancel verb, mirrors BRAND § 6 DO/DON'T column. |

### State copy

| Element | Copy | Trigger |
|---------|------|---------|
| Empty state heading (wp has no cross-family couches yet) | **Just your couch tonight** | Renders above the "+ Add a family" button when `wp.families.length === 1`. |
| Empty state body | *Add another family to share the watchparty.* | Italic Instrument Serif 13px at `--ink-warm`. |
| Soft-cap warning (5th+ family) | **That's a big couch — are you sure?** | Triggers at 5+ families per CONTEXT D-08 + § Specifics; literal copy locked from CONTEXT. `--warn` color, italic Instrument Serif. |
| Hard-cap rejection (8+ — server-side) | **No more room on this couch tonight.** | Toast on CF `resource-exhausted` error (RESEARCH Pattern 2). `--warn` color, NOT `--bad`. |
| Success on add | **Couch added** | Toast at `--good`. Sentence-case, no exclamation. |
| Family-code-not-found error | **No family with that code. Double-check the spelling.** | Toast at `--warn`. Specific failure mode, no leak per RESEARCH § Security ("don't leak whether token is wrong vs too old"). |
| Not-host error (defensive — UI shouldn't reach this) | **Only the host can add families to this watchparty.** | Toast at `--warn`. Surfaces the D-07 host-only authority gate. |
| Already-added (idempotency guard from RESEARCH Pitfall 3) | **The {FamilyName} couch is already here.** | Toast at `--ink-warm` (informational, not warn). |

### Roster collision suffix (D-09)

| Element | Copy | Voice Note |
|---------|------|------------|
| Collision suffix render | **Sam *(Smiths)*** | Plain `Sam` at `--ink`; the `(Smiths)` portion in italic Instrument Serif at `--ink-dim`. Render-time only — never stored in Firestore (mirrors Phase 27 D-04 `(guest)` pattern). Triggers ONLY when collision spans different families per RESEARCH Pitfall 6. |
| Family-display-name source | The existing `families/{code}` doc's `displayName` field, OR a derivation (e.g., pluralized first family member's last name like "Smiths") | Researcher confirms which field. If `displayName` is missing on the foreign family doc, render falls back to the family code itself: `Sam (ABC123)`. Planner picks; UI-SPEC documents the fallback chain. |

### Out-of-scope copy (D-02 guard)

Phase 30 produces ZERO affiliate-related copy, ZERO disclosure surfaces, ZERO monetization-flavored language. The "still free + family-first" voice is preserved by simply not introducing the affiliate surface at all (D-01 carve-out). Any planner or executor who finds themselves writing affiliate disclosure text should STOP — that work belongs to future Phase 30.1.

| Sub-surface | Lives In | Behavior |
|-------------|----------|----------|
| **`.wp-add-family-section`** | wp-create lobby + wp-edit menu | Container wrapping the "+ Add a family" affordance + per-family chip list. Stack-sm vertical rhythm. |

| **`.wp-couches-list`** | Inside `.wp-add-family-section` | Vertical list of family rows. Own family row first (with "(your couch)" suffix), foreign families below. Host sees per-row kebab menu with "Remove this couch" item; non-host sees no kebab. |

### State machines

The "+ Add a family" affordance has 4 visible states:

1. **Idle** — empty input, "Bring them in" submit disabled. `.wp-add-family-input-row` with placeholder text visible.
2. **Validating** — submit pressed, CF `addFamilyToWp` in flight. Submit button shows spinner glyph + label changes to "Bringing them in..." Input disabled.
3. **Success** — toast at `--good` with "Couch added" copy. Input clears. New family row animates into `.wp-couches-list` at `--duration-deliberate` (300ms) per BRAND § 5 Motion table.
4. **Error** — toast at `--warn` with specific copy from the error matrix above. Input retains the bad code so user can correct. Submit re-enabled.

## Interaction Contracts

### Host-only invite authority (D-07)

- The "+ Add a family" affordance is **only rendered** when `state.me.id === wp.hostId`. Non-host members see the `.wp-couches-list` (so they can see who's in the wp) but do NOT see the `.wp-add-family-input-row` or per-row kebabs.
- This is a **render-time gate**, not a styling gate. Don't render the form and disable it; don't render it at all for non-hosts.
- The `addFamilyToWp` CF (RESEARCH Pattern 2) is the server-side trust boundary; the render-time gate is purely UX clarity.

### Soft-cap warning (D-08)

- At `wp.families.length === 4` after a successful add, the "+ Add a family" affordance remains active but the section heading sub-line text **changes** from *"Pull up another family for tonight."* to *"That's a big couch — are you sure?"* (italic Instrument Serif at `--warn`).
- At `wp.families.length === 8` (hard cap from RESEARCH Pattern 2), the "+ Add a family" affordance hides entirely; section heading sub-line changes to *"This couch is full for tonight."* in italic at `--ink-dim`.

### Collision-suffix render (D-09)

- Suffix renders **only** when two participants share the same first name **across different families** (per RESEARCH Pitfall 6). Same name within one family (rare but possible) does NOT trigger suffix.
- Suffix format: ` (FamilyName)` with leading space, in italic Instrument Serif at `--ink-dim`. Never stored in Firestore.
- Pure render-time disambiguation — mirrors Phase 27 D-04 `(guest)` pattern exactly.

### Motion (BRAND § 5)

- New family row entry into `.wp-couches-list`: fade + slide-down at `--duration-deliberate` (300ms) `--easing-out`. Honors `prefers-reduced-motion` clamp.
- Soft-cap warning sub-line text-swap: cross-fade at `--duration-base` (220ms) `--easing-standard`.
- Remove-confirmation modal: backdrop fade `--duration-base` (220ms), sheet slide `--duration-deliberate` (300ms) per BRAND § 7 Tier-3.
- "Couch added" / "No more room" toasts: existing `flashToast` motion contract (no new motion surface).

### Accessibility

- Family-code input: `aria-label="Other family's code"`. `inputmode="text"` (codes are alphanumeric). `autocapitalize="characters"` matches Phase 5 family-join input pattern.
- "+ Add a family" button: native `<button type="button">` so VoiceOver reads "button" not "link."
- Cross-family member chips: same `role="listitem"` already on Phase 27 guest chips at `js/app.js:12358`; suffix is part of the chip's accessible name (no separate aria announcement).
- Remove-confirmation modal: focus traps to the "Keep them" button on open (defensive default — prevents accidental confirm on Enter). Escape key triggers "Keep them."
- Collision suffix italic span: NOT wrapped in `<em>` (which adds semantic emphasis aria-announce); use `<span class="family-suffix">` with `font-style:italic` so screen readers read "Sam Smiths" not "Sam emphasis Smiths." This is the same pattern Phase 27 uses for `(guest)` at `js/app.js:12361`.

### Mobile-first surface (per CLAUDE.md "Test on mobile Safari")

- `.wp-add-family-input-row` collapses input + submit to vertical stack below `600px` viewport.
- "+ Add a family" affordance and family-row kebab menus tap-target ≥ 44px (existing brand rule from BRAND).
- iOS Safari PWA standalone mode: confirm Phase 14's `.detail-close{position:fixed}` rule at `js/app.js` carries to Phase 30 modal close-buttons (no new modal needed; reuses existing `.detail-modal` recipe).
