## Spacing Scale

Declared values (project uses a named scale, not 8pt — mapped below):

| Token | Value | CSS Var | Usage |
|-------|-------|---------|-------|
| xs | 4px | --s1 | Icon gaps, tight inline padding |
| sm | 8px | --s2 | Compact spacing, chip gaps |
| md-sm | 12px | --s3 | Toast horizontal padding, note padding |
| md | 16px | --s4 | Default element spacing, modal inner padding |
| md-lg | 20px | --s5 | Section gaps, button vertical padding |
| lg | 28px | --s6 | Modal title-to-content gap |
| xl | 40px | --s7 | Major section breaks |
| 2xl | 56px | --s8 | Page-level spacing |

Exceptions:
- Touch targets (veto button, spin-result actions): minimum 44px tall on mobile Safari — use `padding: 14px` per existing `.spin-actions button` pattern
- Veto button in spin result modal: full-width, same 14px padding as `.spin-accept` / `.spin-reroll`
- Fairness-disabled Spin button: no extra spacing — same layout, just `opacity: 0.45; cursor: not-allowed`

## Typography

Source: `css/app.css` `:root` block and body declaration. All roles below match the existing type scale exactly.

| Role | Size | CSS Var | Font | Weight | Line Height |
|------|------|---------|------|--------|-------------|
| Body / UI text | 15px | --t-body | Inter | 400 | 1.55 (from html/body rule) |
| Label / meta / eyebrow | 13px | --t-meta | Inter | 500 | 1.4 |
| Heading (modal title, result name) | 17px | --t-h3 | Instrument Serif italic | 400 | 1.2 |
| Display (spin result title card) | 22px | --t-h2 | Instrument Serif italic | 400 | 1.2 |

Weight contract: **400 regular** for body and Instrument Serif display roles; **500 medium** for labels and meta copy; **700 bold** for primary action buttons (per existing `.spin-actions button { font-weight: 700 }`).

Fraunces is a fallback for Instrument Serif — use `font-family: 'Instrument Serif', 'Fraunces', serif` for all heading/brand moments.

## Color

Source: `css/app.css` `:root` — all values verbatim.

| Role | Value | CSS Var | Usage |
|------|-------|---------|-------|
| Dominant (60%) | #14110d | --bg | Page background, modal backdrops |
| Secondary (30%) | #1c1814 / #25201a | --surface / --surface-2 | Modal bodies, cards, toast backgrounds |
| Accent (10%) | #e8a04a | --accent | Reserved elements only (see below) |
| Destructive | #c44536 | --bad | Veto action button bg (post-spin veto — the action removes/blocks a title) |
| Ink (primary text) | #f5ede0 | --ink | Primary copy |
| Ink-warm (secondary text) | #c9bca8 | --ink-warm | Supporting copy, who-vetoed name |
| Ink-dim (tertiary text) | #847868 | --ink-dim | Timestamps, cancel buttons, disabled labels |

**Accent reserved for:**
- The veto count / cap warning text ("you've used both vetoes tonight") — matches existing `--accent` warn tone
- Focus rings on interactive elements (`outline: 2px solid var(--accent)` — already in global rule)
- "Spinning again…" shimmer sweep overlay accent band (inherits `.sk` shimmer pattern)
- Toast border for neutral/informational veto toasts (inherits `.toast` `border-color: var(--border-strong)`)

**Destructive (`--bad`) reserved for:**
- The "Pass on this" / veto submit button in the veto modal — `background: var(--bad)` signals this removes a title
- Cap-exceeded blocked state inline text if needed as a secondary signal

**Good (`--good`, `#7fb069`) reserved for:**
- "Watch it tonight!" / accept button in spin result — no change from existing `.spin-accept`

**Velvet (`--velvet`, `#8b3a4e`) reserved for:**
- Brand gradient moments only — do not introduce for veto UI

## Interaction States

All new components must implement these states consistently.

### Veto Button (in Spin Result Modal)

| State | Visual Treatment |
|-------|-----------------|
| Default | `background: var(--bad); color: white; border-radius: 12px; padding: 14px; font-weight: 700` — full width, same height as `.spin-accept` |
| Pressed | `transform: scale(0.98)` — matches `.spin-cancel:active` |
| Daily cap reached (1st veto used, 1 remaining) | Default — no change |
| Daily cap reached (both vetoes used) | `opacity: 0.45; cursor: not-allowed; pointer-events: none` — same as `.progress-stepper-btn:disabled` |
| Focus | `outline: 2px solid var(--accent); outline-offset: 2px; border-radius: var(--r-sm)` — global rule |

### Spin Button — Fairness Disabled State (VETO-06)

| State | Visual Treatment |
|-------|-----------------|
| Default | Existing spin button styles — unchanged |
| Fairness locked (vetoer's device only) | `opacity: 0.45; cursor: not-allowed` + tooltip text "Someone else spins this one" |
| Tooltip | Appears on tap/hover; use existing toast-style surface: `background: var(--surface-2); border: 1px solid var(--border-mid); border-radius: var(--r-pill); font-size: var(--t-meta); padding: var(--s2) var(--s3)` |

### Veto Note on Title Card (.tc-note.veto) — Existing, No Change

Existing rule applies: `background: var(--bg-2); color: var(--ink-dim)` with `.veto-who { color: var(--ink) }`. Phase 4 extends this display; do not restyle it.

### "Spinning again…" Shimmer State (D-14)

During auto re-spin transition on all devices:
- The spin result card area transitions to shimmer: apply `.sk` class to the poster element
- Duration: 400ms shimmer visible before new flicker sequence begins
- Do not add a new overlay — swap existing `.spin-result-poster` to skeleton then back to flicker sequence

## Copywriting Contract

Source: 04-CONTEXT.md `<specifics>` section + CONTEXT.md decisions. All copy pre-populated from upstream.

| Element | Copy |
|---------|------|
| Veto modal title (pre-spin, from title detail) | "Pass on this one" |
| Veto modal title (post-spin, from spin result) | "Pass on this one" — same title, context implied by location |
| Veto modal description | "Pass on this one just for tonight. Everyone else will see it." (existing, no change) |
| Veto submit button | "Pass on it" |
| Veto cancel / close | "Keep it" (pre-spin) / "Keep it" (post-spin) |
| Veto reason placeholder | "Too long? Wrong mood? Already saw it?" (existing, no change) |
| Real-time veto toast | "[Name] passed on [Title] — spinning again…" — warm, lowercase, em-dash |
| Fairness tooltip | "Someone else spins this one" — soft, not accusatory |
| Daily cap warning (1 veto used, 1 remaining) | No copy change — button stays active |
| Daily cap warning (both vetoes used) | "you've used both vetoes tonight" — retains warm tone, lowercase |
| Empty pool after all relaxations | "Nothing left to spin — try clearing some filters" |
| Re-spin auto-trigger shimmer (no copy shown during shimmer) | — |
| "Vetoed tonight" divider | "Vetoed tonight" (existing, no change) |
| Unveto (undo within session) | "Undo veto" (existing `unveto()` pattern) |

**Destructive actions in this phase:**

| Action | Approach |
|--------|----------|
| Submit veto (removes title from pool) | Single tap on "Pass on it" button — no secondary confirmation. The action is reversible within the session via `unveto()`, so immediate commit is appropriate. Veto modal itself acts as the intent gate. |
| Unveto (undo veto, deletes vetoHistory doc same session) | Single tap on "Undo veto" — no confirmation. Sessions only; after midnight, history is immutable. |

**No other destructive actions in this phase.**

## Animation Contract

| Event | Animation | Duration | Easing |
|-------|-----------|----------|--------|
| Veto modal open | `modal-slide-up` (existing `@keyframes`) | `--t-cinema` (400ms) | `--ease-cinema` |
| Veto modal close | Reverse slide / fade | `--t-base` (220ms) | `--ease-out` |
| Veto toast appear | `opacity: 0 → 1; translateY(8px → 0)` (existing `.toast.on` transition) | `--t-base` (220ms) | default |
| Toast auto-dismiss | Fade out | `--t-base` (220ms) | default |
| Shimmer during re-spin | `.sk` sweep at 1.6s loop | 400ms visible then replaced | `ease-in-out` (existing) |
| Re-spin flicker landing | Existing `spin-land` @keyframes | 400ms | `--ease-cinema` |
| Confetti on accept (re-spin land) | Existing confetti system | Same as initial spin | Same as initial spin |
| Fairness tooltip appear | `opacity: 0 → 1` | `--t-quick` (150ms) | `--ease-out` |

**Reduced motion:** All animations already guarded by `@media (prefers-reduced-motion: reduce)` global rule — no new work needed for Phase 4 components.
