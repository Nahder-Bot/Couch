```html
<!-- Phase 13 / OPS-13-05 — Sentry loader (CDN, zero-bundler).
     Source: docs.sentry.io/platforms/javascript/install/loader/
     PII scrubbed in beforeSend per RESEARCH.md Pattern 4.
     CSP-compatible: js.sentry-cdn.com is allow-listed in queuenight/firebase.json (Plan 13-05).
     DSN is public-by-design (rate-limited per project per docs.sentry.io). -->
<script>
  window.sentryOnLoad = function () {
    Sentry.init({
      dsn: 'https://<SENTRY_PUBLIC_KEY>@o<SENTRY_ORGID>.ingest.us.sentry.io/<SENTRY_PROJECTID>',
      environment: (location.hostname === 'couchtonight.app') ? 'production' : 'dev',
      tracesSampleRate: 0.1,
      replaysSessionSampleRate: 0,
      replaysOnErrorSampleRate: 0.1,
      integrations: [
        Sentry.browserTracingIntegration(),
        Sentry.replayIntegration({ maskAllText: true, blockAllMedia: true })
      ],
      // T-6 mitigation: PII scrub before any network call to Sentry ingest.
      beforeSend: function (event, hint) {
        // 1. Strip user.id (uid), user.email, user.username, ip_address (never off-platform)
        if (event.user) {
          delete event.user.id;
          delete event.user.email;
          delete event.user.username;
          delete event.user.ip_address;
        }
        // 2. Strip family codes (6 lowercase hex chars) from messages + exception values
        var FAMILY_RE = /\b[a-f0-9]{6}\b/gi;
        if (event.message) event.message = String(event.message).replace(FAMILY_RE, '<FAMILY>');
        if (event.exception && event.exception.values) {
          for (var i = 0; i < event.exception.values.length; i++) {
            var ex = event.exception.values[i];
            if (ex && ex.value) ex.value = String(ex.value).replace(FAMILY_RE, '<FAMILY>');
          }
        }
        // 3. Strip tokens from request URLs (?invite= ?claim= /rsvp/<token>)
        if (event.request && event.request.url) {
          event.request.url = String(event.request.url)
            .replace(/[?&](invite|claim)=[^&]+/g, function (_, k) { return '?' + k + '=<REDACTED>'; })
            .replace(/\/rsvp\/[^/?]+/g, '/rsvp/<TOKEN>');
        }
        // 4. Strip same patterns from breadcrumb URLs
        if (event.breadcrumbs && event.breadcrumbs.length) {
          for (var b = 0; b < event.breadcrumbs.length; b++) {
            var bc = event.breadcrumbs[b];
            if (bc && bc.data && bc.data.url) {
              bc.data.url = String(bc.data.url)
                .replace(/[?&](invite|claim)=[^&]+/g, function (_, k) { return '?' + k + '=<REDACTED>'; })
                .replace(/\/rsvp\/[^/?]+/g, '/rsvp/<TOKEN>');
            }
          }
        }
        return event;
      },
      // Pitfall 5 defense: drop Firestore transport noise from breadcrumbs.
      beforeBreadcrumb: function (breadcrumb, hint) {
        if (!breadcrumb) return breadcrumb;
        var msg = breadcrumb.message || (breadcrumb.data && breadcrumb.data.message) || '';
        if (/WebChannelConnection|FirebaseError|firestore\.googleapis/.test(String(msg))) {
          return null;
        }
        return breadcrumb;
      }
    });
  };
</script>
<script src="https://js.sentry-cdn.com/<SENTRY_PUBLIC_KEY>.min.js" crossorigin="anonymous"></script>
```

**Step 1.3 — Sentry SRI hash decision (T-7 mitigation).**

The orchestrator threat list T-7 flagged: "Sentry CDN script tampering. Mitigation: SRI hash + crossorigin=\"anonymous\" on the loader script tag."

**Decision:** Apply `crossorigin="anonymous"` (already in the snippet above). Do NOT add an `integrity="sha384-..."` SRI hash — Sentry's loader script is auto-versioned at the CDN edge ("latest" per project), and adding an SRI hash would lock us to a single immutable build, defeating the auto-update purpose of the loader pattern. The `crossorigin="anonymous"` attribute is the standard Couch posture for cross-origin scripts and provides the per-request integrity envelope that Couch trusts for other CDN imports.

Document this T-7 mitigation choice in `13-02-SUMMARY.md` ("disposition: accept" with the rationale above). The rationale is consistent with how docs.sentry.io describes the loader: "we recommend not pinning a version" — pinning via SRI hash is functionally equivalent to pinning a version.
