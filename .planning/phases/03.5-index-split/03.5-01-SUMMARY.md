# Summary: index.html File Split

## What Was Built

Split the 11,110-line monolithic `index.html` into 6 ES module files with no bundler. Pure structural refactor — no functionality changes.

## Files Created

| File | Lines | Contents |
|------|-------|----------|
| `css/app.css` | 1,888 | All CSS (extracted from inline `<style>`) |
| `js/firebase.js` | 17 | Firebase CDN imports, config, `db` init; exports all |
| `js/constants.js` | ~130 | TMDB_KEY, Trakt config, COLORS, RATING_TIERS, MOODS, normalizeProviderName, etc. |
| `js/state.js` | 6 | `state` object, `titlesRef`, `membersRef`, `familyDocRef` |
| `js/utils.js` | ~65 | `escapeHtml`, `haptic`, `flashToast`, poster helpers |
| `js/app.js` | 8,145 | All feature logic; imports from the above |

## Files Modified

- `index.html`: reduced from 11,110 → 821 lines (HTML shell + `<script type="module" src="js/app.js">`)
- `CLAUDE.md`: updated architecture section to reflect modular layout

## Token Cost Impact

| Before | After |
|--------|-------|
| 145K tokens to read any JS | index.html: ~10K tokens |
| CSS edit = 145K context | css/app.css: ~23K tokens |
| Any change = full file read | firebase/constants/state/utils: <3K each |

## Issues Encountered

- `fetchTmdbExtras` closing `}` was off-by-one in range extraction (line 948 not 947) — caused `SyntaxError: Unexpected end of input`. Fixed immediately after browser test.
