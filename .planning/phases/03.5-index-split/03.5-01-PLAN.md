# Phase 3.5 Plan 01: index.html File Split

## Objective

Split the 11,110-line index.html into per-feature ES module files with no bundler. Use explicit `import/export` between files. Keep all `window.xxx =` onclick handlers as-is. Keep all current functionality exactly unchanged.

**Goal:** Future agents editing veto/watchparty/year-review code read ~15-30K tokens instead of 145K.

## Critical Constraints

- **NO bundler, NO build step.** Firebase Hosting serves files directly.
- **NO functionality changes.** Pure structural refactor.
- **index.html is 580KB (~145K tokens). NEVER read it in full.** Use only targeted line reads:
  - `Read(file, offset=N, limit=100)` — read specific ranges
  - `Grep` to locate functions, then read ±30 lines around them
- **ES module imports/exports.** One entry `<script type="module" src="js/app.js">` in index.html.
- **Test with `firebase serve`** after completing the split. Verify app loads and functions.

## Export Rules

1. **`window.xxx =`** assignments: keep exactly as-is in whatever file they move to. Do not change onclick handlers.
2. **Functions called by other files**: add `export` keyword before `function` or `const`.
3. **Functions only called within their own file**: no `export` needed (stay private).
4. **`state`, `db`, Firestore functions**: must be exported from their source file and imported everywhere used.
5. **`MOODS`, `TMDB_KEY`, etc.**: export from constants.js, import where needed.
