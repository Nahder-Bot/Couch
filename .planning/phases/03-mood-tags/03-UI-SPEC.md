## Spacing Scale

The project uses a custom `--s{N}` token scale (not 8-point). All new elements must reference these tokens exclusively — never raw pixel values.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| --s1 | 4px | Icon-to-label gap inside a mood chip |
| --s2 | 8px | Gap between chips in a chip row; chip row gap |
| --s3 | 12px | Internal margin between mood section and adjacent detail sections |
| --s4 | 16px | Horizontal chip padding (matches existing `.mood-chip` padding) |
| --s5 | 20px | Vertical margin above mood section in detail view |
| --s6 | 28px | Not used in this phase |
| --s7 | 40px | Not used in this phase |

Exceptions:
- Touch target for the × remove button on active-filter chips: minimum 36×36px tap area (achieved by padding the × within the chip element, not by changing chip height). The × itself renders at `--t-meta` (13px).
- The "+" add-mood affordance in the detail view: renders as a `.mood-chip`-shaped element — same height as other chips — to preserve visual unity.

Source: existing `:root` tokens; confirmed by `.mood-chip` definition at line 628.

## Component Inventory

### 1. Detail View — Mood Section

**Location:** Between `.detail-genres` (genre pills row) and `.detail-overview` (synopsis paragraph), inside `.detail-body`.

**Container class:** `.detail-moods` (new)

```
.detail-moods {
  display: flex;
  flex-wrap: wrap;
  gap: var(--s2);
  margin-bottom: var(--s4);
}
```

**Section label (optional):** A `.detail-section h4` eyebrow ("Moods") above the chip row, consistent with other detail section labels (line 1322: `font-family:Inter, font-size:--t-micro, color:--accent, text-transform:uppercase, letter-spacing:0.18em, font-weight:600`). Include label only if the section has chips or is in empty state — suppress the label entirely if it would appear alone with nothing beneath it.

**Chip rendering:** Reuse `.mood-chip` as-is. Each chip for a title that has moods renders in the off-state (no `.on` class — these are not "active filter" chips, they are "present on title" chips). Tapping/clicking a chip removes it via `arrayRemove`.

**Empty state:** A single chip-shaped affordance renders when `title.moods` is empty. Style: same `.mood-chip` dimensions and pill shape, but uses dashed border `1px dashed var(--border-strong)`, background `transparent`, color `var(--ink-dim)`. Label: `+ add mood`. Tapping it opens the mood picker (inline palette, not a modal).

**"+" Add affordance:** When moods exist on the title (non-empty), the add affordance renders as a trailing chip in the same row. Style: same as empty-state chip above (`+ add mood`, dashed border). Tapping reveals the inline palette.

**Mood picker palette:** Renders inline as a second wrapped row beneath the existing chips — not in a modal, not in a bottom sheet. Shows all 10 moods from `MOODS[]` that are not already on the title. Each palette chip uses `.mood-chip` without `.on`, plus `background: var(--surface-2)` to distinguish from the title's own chips. Tapping a palette chip adds it via `arrayUnion` and dismisses the palette. Tapping outside the palette (document click) or re-tapping the "+" chip dismisses without adding.

### 2. Tonight Filter Bar — Active Mood Inline Row

**Location:** A new `<div id="t-mood-active-row">` inserted immediately after the `.t-filters` flex row (outside `.t-filter-body`), before the main title list. Visible only when `state.filters.moods.length > 0`.

**Container class:** `.t-mood-active-row` (new)

```
.t-mood-active-row {
  display: flex;
  gap: var(--s2);
  flex-wrap: wrap;
  padding: var(--s2) 0 var(--s1);
}
```

**Per-chip structure:** Each active mood renders as a `.mood-chip.on` with an appended `<button class="mood-chip-remove">` inside it. The × button:
- Content: `×` character (U+00D7) or an `aria-label="Remove [mood name]"` span
- Style: `background: transparent; border: none; color: var(--bg); font-size: var(--t-meta); cursor: pointer; padding: 0 0 0 var(--s1); line-height: 1; opacity: 0.75;`
- Hover: `opacity: 1`
- Tap/click: removes that mood from `state.filters.moods`, updates the spin candidate pool, re-renders this row and the count badge

**Count badge:** Existing `.t-filter-count` on the "Mood" toggle pill continues to show the count (e.g., "2") and the `.t-filter-toggle.has-active` accent fill continues as-is. No change to existing badge behavior.

**Row visibility:** When `state.filters.moods` empties (all moods removed), hide this row (`display: none` or remove from DOM). No animation needed.

## Interaction States

### Mood chip in detail view

| State | Visual |
|-------|--------|
| Default (mood present on title) | `.mood-chip` off-state — surface bg, ink-warm label, mid border |
| Hover | `border-color: var(--border-strong); color: var(--ink)` (existing `.mood-chip:hover`) |
| Active/press | `transform: scale(0.97)` — 150ms ease-out |
| Removing (optimistic) | Chip fades out at opacity 0 in 150ms, then removed from DOM |
| Focus-visible | `outline: 2px solid var(--accent); outline-offset: 2px` (global rule) |

### "+" add-mood affordance

| State | Visual |
|-------|--------|
| Default | Dashed border, `var(--ink-dim)` label, transparent bg |
| Hover | `border-color: var(--accent); color: var(--ink)` |
| Active (palette open) | `border-color: var(--accent); color: var(--accent)` |

### Active filter chip with × (Tonight row)

| State | Visual |
|-------|--------|
| Default | `.mood-chip.on` — accent bg, bg-color label, shadow-warm |
| × hover | `opacity: 1` on × glyph |
| Chip removed | Row re-renders without this chip (no animation required) |

### Transition timing

All chip interactions use `transition: all var(--t-quick)` (150ms) consistent with existing `.mood-chip` declaration (line 633).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (add affordance) | `+ add mood` |
| Empty state heading | *(none — no heading needed; the add chip is the affordance)* |
| Empty state chip label | `+ add mood` |
| Mood removal (detail chip tap) | No confirmation — remove is immediate and reversible by re-adding. No destructive dialog. |
| Active filter × button aria-label | `Remove [mood label]` (e.g., "Remove Cozy") |
| Mood palette dismiss | No copy — tap outside or re-tap "+" to dismiss |
| Error state (Firestore write failure) | Not surfaced in UI for this phase — write failures are silent (optimistic update reverts on next real-time sync). No error toast required. |

Rationale for no-confirmation removal: mood removal is non-destructive — any family member can re-add the mood immediately. No ownership model (D-02). Confirmation would add friction inconsistent with the "always-editable" principle (D-05).

Source: CONTEXT.md D-05 (always-editable, no toggle), D-02 (no ownership).

## Accessibility Contract

- All `.mood-chip` elements that are interactive must have `role="button"` and `tabindex="0"` if rendered as `<span>` or `<div>`. Prefer `<button>` elements.
- The × remove button inside active-filter chips must be a `<button>` with `aria-label="Remove [mood name]"`.
- The mood palette (when open) must be reachable by keyboard tab sequence immediately after the "+" button.
- Focus trap is not required — the palette is inline, not a modal.
- Touch targets: minimum 36×36px tappable area for all interactive chips (achieved via padding, not min-height, to preserve visual compactness).
- The inline active-mood row must have `aria-label="Active mood filters"` on its container.
