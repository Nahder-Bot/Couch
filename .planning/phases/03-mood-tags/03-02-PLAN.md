function renderMoodFilter() {
  const el = document.getElementById('mood-filter-tonight');
  if (!el) return;
  // ... available mood computation from state.titles ...
  const chips = shown.map(m => {
    const on = state.selectedMoods.includes(m.id);
    return `<button class="mood-chip ${on?'on':''}" onclick="toggleMood('${m.id}')">

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser DOM → Tonight filter state | `removeMoodFilter(id)` accepts a string id from onclick. In production, ids are sourced from `state.selectedMoods`, which is only populated by `toggleMood` (which itself receives from MOODS[]). A modified page could call `removeMoodFilter('<script>')` directly. |
| updateFiltersBar → DOM | Renders mood labels into innerHTML. Labels come from `MOODS[]` constant (trusted), but escaped for defense-in-depth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | `window.removeMoodFilter(id)` | mitigate | `if (!moodById(id)) return;` gate — rejects any id not in `MOODS[]` before touching `state.selectedMoods` or calling `renderTonight`. |
| T-03-08 | Tampering (XSS) | updateFiltersBar active-row template | mitigate | All interpolated values (mood id, label) pass through `escapeHtml()`. Mood id AND label are escaped in the onclick AND aria-label strings. Mirrors existing `renderMoodFilter` convention (line 4888). |
| T-03-09 | Tampering | `state.selectedMoods` arbitrary injection via `toggleMood('<script>')` | accept | `toggleMood` currently has no moodById guard. This is out of scope for Phase 3 (existing Phase 2 code). If exploited, the render path still calls `moodById(m.id)` in both `renderMoodFilter` (line 4882) and the new active-row code — invalid ids are silently skipped, so no XSS payload reaches innerHTML. No mitigation required here. |
| T-03-10 | Denial of Service / UX | Active-row innerHTML flicker during unrelated Tonight re-renders | mitigate | `if (activeRow.innerHTML !== html) activeRow.innerHTML = html;` guard (Pitfall 4). Prevents needless layout. Observable benefit only; no security impact. |
| T-03-11 | Tampering (HTML parse) | Nested-button structure (active-row chip containing × button) | mitigate | Outer wrapper is `<div class="mood-chip on" role="group">`, × is the only `<button>`. Pitfall 2 — prevents Safari input-handling quirks and invalid HTML. No silent violations possible because grep-based acceptance criteria verify the outer is `<div`. |
</threat_model>
