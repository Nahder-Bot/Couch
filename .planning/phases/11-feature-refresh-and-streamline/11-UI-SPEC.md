## Spacing Scale

Declared values — project uses named scale (`--s1`…`--s8`) with semantic aliases. Extracted verbatim from `css/app.css:62-70`. All multiples of 4. **No new spacing tokens introduced this phase.**

| Token (primitive) | Value | Semantic alias | Usage |
|-------|-------|----------------|-------|
| `--s1` | 4px | `--space-inline-xs` | Icon gaps, tight inline padding, mood-chip gap (after REFR-01 tighten) |
| `--s2` | 8px | `--space-inline-sm` | Compact spacing, mood-chip gap (current → tightened) |
| `--s3` | 12px | `--space-inline-md` · `--space-stack-sm` | Toast padding, vertical rhythm inside cards |
| `--s4` | 16px | `--space-inline-lg` | Default element spacing, modal inner padding |
| `--s5` | 20px | `--space-stack-md` | Section gaps, button vertical padding, `.who-card` padding |
| `--s6` | 28px | `--space-stack-lg` | Tab-section separator gap, modal title-to-content |
| `--s7` | 40px | `--space-section` | Major section breaks |
| `--s8` | 56px | `--space-page` | Page-level breathing (hero → content) |

**Exceptions (phase-specific):**
- **Touch targets — 44px minimum** on mobile Safari (iOS PWA is primary surface). Applies to: Ready-check toggle, Sports DVR slider thumb, post-session rating stars, RSVP page Yes/Maybe/No buttons, "Catch me up" dismiss, "Schedule next" CTA. Enforce via `min-height: 44px` (button) or `padding: 14px` (per existing `.spin-actions button` pattern at css/app.css).
- **Mood chip density tightening (REFR-01):**
  - `.mood-filter` gap: `var(--s2)` (8px) → `var(--s1)` (4px)
  - `.mood-chip` padding: current `var(--s2) var(--s3)` (8px 12px) → `var(--s1) var(--s3)` (4px 12px)
  - Tap target preserved via `min-height: 36px` on `.mood-chip` (below the 44px main-CTA floor; acceptable for chip UI per existing wp-control-btn precedent of 32-36px height)
- **Tab restructure density (REFR-11/12):**
  - Section-to-section gap inside `#screen-family` and `#screen-settings`: keep existing `var(--s6)` (28px) from `.tab-section { margin-top: var(--s6) }` — no change.
  - Sub-cluster headings (Account tab's 3 cognitive clusters — You / Couch-wide / Admin) use `.tab-section-h` at 11px uppercase eyebrow + `var(--s3)` (12px) bottom margin before first child card.
- **Sports score strip (REFR-10):** Fixed-position chrome at top of `.wp-live-modal` body — height `40px`, internal padding `var(--s2) var(--s3)` (8px 12px). 40px is an existing scale value (`--s7`? No — 40px = `--s7`; this is an s7-divided-by-1-exception; acceptable as a fixed chrome height since other fixed chromes like `.tabbar` already use fixed heights in the 64-80px range).
- **Lobby countdown ring (REFR-07):** 56px diameter (`--s8`) for the T-minus timer badge at top of lobby state. Existing `--s8` token covers this without introducing new spacing.

**Weight contract (BRAND.md §3):**
- **400 regular** for all Inter body/UI copy AND all Fraunces/Instrument Serif display roles.
- **500 medium** for meta + micro (subtle emphasis).
- **600 semibold** for eyebrows + secondary buttons.
- **700 bold** for primary action buttons ONLY (existing `.spin-actions button`, `.wp-banner-action`, new `.wp-ready-check`, new `.wp-post-session-cta`).

**Anti-pattern (BRAND.md §3):** Do NOT mix Fraunces and Instrument Serif in the same visual section. Pick one per section and hold the line.
- Lobby + post-session modals: Instrument Serif for all italic accents; Inter for body.
- Sports game picker + score strip: Fraunces for display (title); Inter for score numerals (monospace feel via tabular-nums — `font-variant-numeric: tabular-nums`).
- RSVP standalone page (REFR-05): Fraunces wordmark at top ONLY; Instrument Serif for the RSVP page tagline; Inter for all form labels/buttons. Lean cinematic, not app-shell-heavy.

**Tabular numerals required for:**
- Sports score strip (home/away score, quarter/period, clock)
- Lobby countdown (T-15:00 → T-00:00 — avoid digit-width jitter)
- DVR offset readout ("I'm 12s behind")
- Watchparty elapsed-time chips (already implement tabular — extend the pattern)

### REFR-03 Who's-on-the-couch card — 3 variants for planner to choose from

Planner picks ONE of these at plan-phase. All three share spacing (`padding: var(--s5)`) and surface (`var(--surface)` with `border-radius: var(--r-lg)`).

| Variant | Layout | Pros | Best for |
|---------|--------|------|----------|
| **A. Overlapping avatar stack** | Horizontal row, avatars at 44px overlapping -12px, name-strip below avatars as italic Instrument Serif 17px comma-separated | Dense, cinematic, "ensemble cast" feel | Smaller families (2-5 members) |
| **B. Compact horizontal row** | Horizontal scrollable chip row, each chip `.wp-participant-chip`-pattern (26px avatar + inline name), fits 6-8 per viewport | Scales gracefully to large families, familiar pattern from watchparty | Larger families (5+ members), visual consistency with lobby strip |
| **C. Per-person compact rows** | Vertical list, each row 40px tall: avatar left + name + tiny "last watched" meta right | Information-rich, supports drill-down, feels intentional | Power users who want glanceable "who's around" signal |

**Default recommendation:** Variant B (compact horizontal row) — reuses existing `.wp-participant-chip` CSS, lowest risk, matches the density direction of REFR-01/02. Planner may override at plan-phase based on mockup preference.
