## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Browser PWA → Firestore (rules) | Untrusted client writes (memberId, tupleKey) cross here; rules enforce per-key isolation | progress map slices, tupleNames inner keys, coWatchPromptDeclined inner keys, actingTupleKey echo |
| Source rules → Mirror rules | Code-review boundary — drift means tests pass but prod runs different code | `firestore.rules` (couch repo source vs `~/queuenight` deploy mirror) |
| validAttribution() helper → All authed writes | Single chokepoint for memberId/managedMemberId anchor; Phase-1-14 paths inherit the new check | memberId, managedMemberId, actingUid |
| Couch repo deploy → Production hosting | scripts/deploy.sh propagates code; CACHE bump triggers PWA invalidation | js/app.js, sw.js, changelog.html, css/* |
| Queuenight repo deploy → Production rules | firebase deploy --only firestore:rules propagates rules; takes effect immediately on the running Firestore instance | firestore.rules |
