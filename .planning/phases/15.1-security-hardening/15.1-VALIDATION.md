> Retrospective Nyquist audit. Phase shipped to production 2026-04-27 (v35.1); this document reconstructs the validation contract and closes two coverage gaps surfaced post-hoc.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `@firebase/rules-unit-testing` 3.0.2 + Firebase Emulator (firestore-only) |
| **Config file** | `tests/firebase.json` + `tests/package.json` |
| **Quick run command** | `cd tests && npm test` |
| **Full suite command** | `cd tests && npm test` (single-suite project — no quick/full split) |
| **Estimated runtime** | ~15–25 seconds (emulator boot + 40 rules-unit tests) |

The `pretest` hook in `tests/package.json` copies `../firestore.rules` → `tests/firestore.rules` before each run, so the suite always pins the source rules file (the deploy mirror at `~/queuenight/firestore.rules` is checked separately for byte-identity, not for test inclusion).

The project deliberately has no JS/UI/browser test framework — CLAUDE.md locks the architecture as "no bundler — Firebase Hosting serves files directly," and adding Vitest/Jest+jsdom is out of scope for this phase. UI requirements (e.g. SEC-15-1-05 render gates) are covered manually; see Manual-Only Verifications below.
