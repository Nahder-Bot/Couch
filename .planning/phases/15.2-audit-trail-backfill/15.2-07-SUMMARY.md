## Open Questions / Future Work

### init reader does not read STATE.md milestone slug (RESEARCH Open Question #1, confirmed empirically)

- **Symptom:** After commit, `node $HOME/.claude/get-shit-done/bin/gsd-tools.cjs init progress` still returns `"milestone_version": "v33.3"` and `"milestone_name": "milestone"` despite STATE.md frontmatter now showing `milestone: v1-commercial-release` + `milestone_name: v1 Commercial Release`.
- **Root cause:** The init reader's `getMilestoneInfo` (in `$HOME/.claude/get-shit-done/bin/lib/core.cjs:1505-1540`) reads ROADMAP.md and matches a regex `v(\d+(?:\.\d+)+)` that requires a dotted version (e.g., `v1.2`, `v1.2.1`). The canonical `v1-commercial-release` slug is kebab-case with NO dots, so it doesn't match. The reader's fallback regex finds the FIRST occurrence of any `v\d+\.\d+\d` pattern in ROADMAP.md — currently `v33.3` from line 246 (inside Phase 15.2's success-criteria block reference to `.planning/v33.3-MILESTONE-AUDIT.md`).
- **Impact:** `init progress` / `init phase-op` outputs show `milestone_version: v33.3` instead of canonical slug. Branch templates that interpolate `{milestone}` (per config.json `milestone_branch_template: "gsd/{milestone}-{slug}"`) would produce stale prefixes. STATE.md frontmatter is internally consistent and the canonical source of truth; only the init reader's view is degraded.
- **Plan posture:** The plan explicitly flagged this as RESEARCH Open Question #1 and declared it non-blocking: "If `gsd-tools.cjs status` or `init.phase-op` return errors, document in plan execution log — the init reader's behavior with kebab-case milestone slugs is per RESEARCH Open Question #1; degraded experience is non-blocking for this plan's must-haves." Confirmed — must-haves are met regardless.
- **Future fix (out of scope):** Teach init reader's `getMilestoneInfo` to prefer STATE.md frontmatter `milestone:` + `milestone_name:` fields when present, falling back to ROADMAP.md regex only when STATE.md is absent or omits the fields. Estimated 20-line change in `core.cjs:1505-1540`. Could be filed as a get-shit-done framework PR, separate from Couch project work.
