<objective>
Give each user per-event-type opt-in/out control with server-side enforcement. Client writes toggles to `users/{uid}/notificationPrefs`; CF `sendToMembers` receives an `eventType` parameter and skips recipients whose prefs for that type are `false`. This satisfies PUSH-02 granular opt-in.

Purpose: Forces Couch's push UX into the per-event-granular pattern validated by every competitor in 06-RESEARCH.md (Plex, Kast-minus, Letterboxd, TV Time). Defaults-on for the 4 common events and defaults-off for the 2 potentially-noisy events (veto-cap, tonight's-pick) means users get reasonable behavior out of the box without training them that Couch is noisy.
Output: Settings screen gains a toggle list; Firestore user doc gains `notificationPrefs` map; CF `sendToMembers` reads+respects the map; existing 3 triggers pass their `eventType` string when calling.
</objective>
