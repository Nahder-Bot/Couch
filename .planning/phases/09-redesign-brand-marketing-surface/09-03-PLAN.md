<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Static HTML (index.html) → CSS (app.css) | Style resolution via class attribute matching |
| Static HTML (data-attrs, id=) → Dynamic JS (js/app.js) | JS reads element IDs + classes; assumes certain inline styles present as initial state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | Denial of Service | JS `.style.display = 'block'` fails to override a `.is-hidden !important` class | mitigate | Explicit constraint: do NOT migrate any screen-section or modal-bg display toggle to a class. All 20+ .style.display write sites in js/app.js keep working because the inline style= attribute they toggle stays in place. |
| T-09-03-02 | Tampering | Class-name collision with existing class accidentally changes a different element | mitigate | New class names prefixed semantically (brand-hero-*, page-*, signin-*, text-*, cluster, stack) — grep each new class name against baseline CSS before adding to confirm no existing rule uses it. |
| T-09-03-03 | Information Disclosure | Class names leak design-system semantics to page source | accept | Rationale: CSS is public-by-design. No secrets. |
| T-09-03-04 | Denial of Service | Inline-style purge misses a font-family fallback chain like `font-family:'Instrument Serif','Fraunces',serif` | mitigate | Verify regex matches both quoted and unquoted primary-font names AND the fallback chain syntax. Grep broadens to catch both declarations and also font-style:italic as an adjacent smell. |
| T-09-03-05 | Tampering | Dynamic transform style (e.g. `transform:rotate(Xdeg)` for picker spin) accidentally migrated to a static class | mitigate | Case-by-case PRESERVE rule on dynamic transforms. No class exists for "rotate by arbitrary degrees"; inline style stays. |
| T-09-03-06 | Denial of Service | Class consolidation in one commit drops an expected className and later JS write fails silently | mitigate | Pre-commit class-presence census (Major-5 checker fix). Every task re-runs the Step-A grep and HALTS if any tracked class count drops unexpectedly. Baseline values recorded in SUMMARY + commit body for audit. |

</threat_model>
