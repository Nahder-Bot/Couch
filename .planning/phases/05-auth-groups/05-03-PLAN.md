<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser тЖФ Cloud Function | Firebase Callable auto-verifies ID token in `req.auth` тАФ no manual JWT parsing |
| Cloud Function тЖФ Firestore | Admin SDK bypasses rules тАФ every security invariant must be coded IN the function body |
| `passwordHash` in Firestore тЖФ client | Rules deny client reads of `families/*` writes; only `onCall` тЖТ admin write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Spoofing | Forged caller identity | mitigate | `onCall` auto-verifies Firebase ID token тЖТ `req.auth.uid` is trusted. Every function throws `HttpsError('unauthenticated')` when `req.auth` missing |
| T-05-03-02 | Info-disclosure | `passwordHash` leak to client | mitigate | Rules deny `families/*` writes from clients (Plan 04); hash never returned from any CF; bcrypt cost 12 makes offline brute-force ~250ms/attempt |
| T-05-03-03 | Tampering | Replay of claim token on multiple devices | mitigate | `claimMember` runs `runTransaction` on `consumedAt` flag; second attempt throws `already-exists` |
| T-05-03-04 | Tampering | Replay of guest invite link | mitigate | Same pattern тАФ invite docs have `consumedAt + status`; `consumeInviteToken` transaction enforces single-use |
| T-05-03-05 | Elevation-of-privilege | Non-owner calls `setGroupPassword` or `transferOwnership` | mitigate | Both functions read `fam.ownerUid` and compare to `req.auth.uid`; throw `permission-denied` otherwise |
| T-05-03-06 | Elevation-of-privilege | Transfer ownership to sub-profile | mitigate | `transferOwnership` verifies target member doc has matching `uid` field; sub-profiles have `managedBy` but no `uid` тЖТ `failed-precondition` |
| T-05-03-07 | Denial-of-service | Brute-force password attempts on `joinGroup` | mitigate | bcrypt cost 12 = ~250ms/call тЖТ ~4 attempts/sec/instance; `maxInstances: 10` in CF config caps concurrency; Phase 6 will add per-family rate limits (deferred per RESEARCH ┬з4) |
| T-05-03-08 | Repudiation | Who changed ownership | mitigate | `transferOwnership` writes `previousOwnerUid + ownershipTransferredAt` onto family doc as immutable log |
| T-05-03-09 | Tampering | Malicious client inflates `durationMs` on `inviteGuest` | mitigate | `inviteGuest` validates `durationMs` is in the allowed set (or тЙд 90-day max); throws `invalid-argument` otherwise |
| T-05-03-10 | Info-disclosure | Invite-token enumeration | mitigate | 32 random bytes base64url = ~256 bits entropy; unguessable; Firestore rules (Plan 04) block client reads of `invites/*` subcollection |
</threat_model>
