<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client Firestore SDK ↔ Firestore backend | Rules evaluate every read/write; admin SDK (in CF) bypasses rules |
| `/settings/auth` ↔ rule helper `graceActive()` | Rules `get(...)` the doc each evaluation (cached per-request) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Elevation-of-privilege | Unauthed client writes to any family doc | mitigate | Every match block's write requires `signedIn()` via helper chains; unit test #6 verifies |
| T-05-04-02 | Elevation-of-privilege | Authed but non-member writes to someone else's family | mitigate | `isMemberOfFamily(code)` helper queries `users/{uid}/groups/{code}` existence; unit test #3 verifies |
| T-05-04-03 | Tampering | User forges `actingUid` claim in payload | mitigate | `validAttribution()` asserts `request.resource.data.actingUid == request.auth.uid`; test #3 verifies deny |
| T-05-04-04 | Tampering | User forges `managedMemberId` for a sub-profile they don't manage | mitigate | `validAttribution()` does a `get(.../members/$(managedMemberId)).data.managedBy == uid()` check; can be added as test #9 |
| T-05-04-05 | Tampering | Post-grace client forges legacy `memberId` to bypass uid checks | mitigate | Rule flips `graceActive()` branch off once `settings/auth.graceUntil < request.time`; test #5 verifies |
| T-05-04-06 | Info-disclosure | Client reads `families/{code}/invites/{token}` to enumerate | mitigate | `invites` match block `allow read, write: if false`; test #7 verifies |
| T-05-04-07 | Info-disclosure | Client reads `families/{code}` directly to extract passwordHash | mitigate | Family doc `read` gated to `isMemberOfFamily`; member read returns passwordHash but: (a) it's bcrypt cost 12 — offline attack ~250ms/attempt, (b) TODO: Plan 07 scrubs passwordHash from member-facing reads via CF-returned family snapshot if threat model expands |
| T-05-04-08 | DoS | Rule expression complexity triggering Firebase's 10-lookup cap | accept | Helpers above average 2-3 lookups per request. Grace branch adds 1. Still under cap; monitored post-launch |
| T-05-04-09 | Repudiation | No audit of who wrote what | accept | `actingUid` field IS the audit log; Firestore doesn't deliver a separate audit surface without extra cost (deferred) |
</threat_model>
