  <name>Task 1: iOS standalone PWA provider UAT — Google, Apple, Email link, Phone</name>
  <what-built>
    Plans 01/02/06 collectively deliver the sign-in flow. iOS PWA redirect has a history of breaking; this checkpoint forces physical-device verification per RESEARCH §9.
  </what-built>
  <how-to-verify>
    Required hardware: physical iPhone running iOS 17 or later, plus either (a) a second iOS device or (b) an Android + desktop Chrome. One real phone number for SMS, one email account.

    **Fresh-install path:**
    1. On iPhone: open Safari → `https://couchtonight.app` → Share → Add to Home Screen → Add.
    2. Open PWA from home screen (standalone mode, no Safari chrome visible).
    3. Sign-in screen appears.

    **Google provider:**
    4. Tap "Continue with Google" → redirects to Google auth → complete sign-in → returns to PWA → DOM shows authenticated UI (family-screen or group-switcher).
    5. Force-quit PWA (swipe up + up on app card). Reopen → still signed in (bootstrapAuth restores auth.currentUser).
    6. Result: PASS / FAIL (+ error details if fail).

    **Apple provider:**
    7. Sign out from settings. Sign-in screen reappears.
    8. Tap "Continue with Apple" → redirects to Apple → complete (use Face ID or password) → returns to PWA → authenticated.
    9. First-time Apple: user name captured (check Firestore user doc or state.auth.displayName).
    10. Result: PASS / FAIL.

    **Email link:**
    11. Sign out. Tap "More options". Enter email. Tap "Send sign-in link." Toast appears.
    12. Open email client → email arrived within ~2min → tap magic link → opens PWA → authenticated.
    13. Test variant: open the link inside Gmail in-app browser vs Mail.app. Document both outcomes.
    14. Result: PASS / FAIL (note which mail client worked).

    **Phone SMS:**
    15. Sign out. More options. Enter phone in E.164 format. Tap "Send code." Invisible reCAPTCHA resolves (may briefly show challenge if suspicious). SMS arrives.
    16. Enter 6-digit code. Tap Verify. Authenticated.
    17. Result: PASS / FAIL.

    **Report:**
    For each provider: PASS/FAIL, any console error messages, round-trip latency (approximate).
    If a provider FAILS, return the error to the executor for a fix pass on Plan 02 (external config) or Plan 06 (client code) before continuing to Task 2.
