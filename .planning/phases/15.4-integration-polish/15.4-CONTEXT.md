### F-W-1 — sendCouchPing fan-out path
- **D-01:** Take **Path A** — wire real push delivery for couch-ping. Reasoning:
  - User just shipped Phase 15.5 (cross-repo CF + client mirror) and has the muscle memory for the deploy ritual
  - Path B (remove the toast lie) would silently retire a DECI-14-06 promise; user-visible regression
  - Path A closes both F-W-1 AND DECI-14-12 push catalog completion — better leverage per unit of work
- **D-02:** New event type: `couchPing` (matches existing camelCase in NOTIFICATION_DEFAULTS)
- **D-03:** Server-side fan-out lives in a **new dedicated CF** `onCouchPingFire` triggered by Firestore writes to a new ephemeral collection `families/{familyCode}/couchPings/{pingId}` (rather than overloading `onIntentCreated`). Reasoning: couch-pings have different semantics (ephemeral, no persistent intent doc, no counter-time, no auto-convert) and bundling into intents breaks the existing intent invariants
- **D-04:** Client-side: `sendCouchPing()` writes a doc to `couchPings`. CF reads sender + recipient name + family context, fans out to recipient's FCM tokens with body "{Sender} wants you on the couch tonight" (final copy locked at planning)
- **D-05:** Self-echo guard: never push to sender's own FCM tokens (per Phase 6 pattern via excludeUid)
- **D-06:** Quiet-hours respected via existing `isInQuietHours` helper (Phase 6 baseline)
- **D-07:** Sentry breadcrumb (Path C from 14-10) replaced by an actual delivery-attempt log on the CF side; the client-side breadcrumb stays as analytics signal
