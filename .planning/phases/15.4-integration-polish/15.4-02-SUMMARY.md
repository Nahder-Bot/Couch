### Task 1 — sendCouchPing Path C → Path A rewrite (commit `31594fd`)

Replaced the entire Path C stub at js/app.js:14477-14506 (29 lines) with a 60-line Path A async implementation:

- **Function signature** changed from `function(memberId)` to `async function(memberId)`. Existing call sites at js/app.js:14399 (V5 long-press resolved-out branch) and js/app.js:14562 (couchMarkAllOut bulk-fire helper) do NOT await the return value, so the signature change is non-breaking.
- **Self-echo short-circuit** added client-side: `if (memberId === state.me.id) return;`. Defense-in-depth on top of the rules-side `senderId == memberId` clause + the CF-side `excludeMemberId` in sendToMembers. Saves a Firestore round-trip in the self-echo edge case (vs falling through to the addDoc which the rules would then deny).
- **Sentry breadcrumb retained** at `category: 'couch-ping'` per CONTEXT D-07 — the only client-side product-analytics signal for ping volume even now that the real fan-out is wired. Message changed from `'would-fire ${memberId}'` (Path C semantic) to `'fire ${memberId}'` (Path A semantic).
- **flashToast retained** at the bottom of the function — copy `'Push sent to ${recipientName}'` no longer "lies" because the push IS being delivered (modulo recipient `notificationPrefs[couchPing]` toggle + quiet hours, both enforced server-side per the Phase 6 baseline).
- **Path A doc shape:** `{senderId: state.me.id, senderName: state.me.name || 'A family member', senderUid: state.me.uid || null, recipientId: memberId, createdAt: Date.now(), ...writeAttribution()}`. The `writeAttribution()` spread stamps `actingUid` and `memberId` (= state.me.id, or `managedMemberId` for proxy writes) which the rules `validAttribution()` helper pins to `request.auth.uid`. Combined with the rules `senderId == memberId` clause, this transitively pins `senderId == request.auth.uid` — Member A cannot write a ping claiming to be from Member B.
- **Defensive try/catch** wraps the addDoc call only; the flashToast sits after the catch block so the user always gets confirmation. On write failure: `console.warn('[15.4/F-W-1] sendCouchPing write failed:', e && e.message)` plus `Sentry.captureException(e, { tags: { area: 'couch-ping' } })` for post-incident debuggability. Trade-off documented in `key-decisions` above.
- **Banned-words sweep** on the new comments + retained copy: PASS (no new copy strings introduced — flashToast template + Sentry message unchanged from Path C / Plan 14-10's already-audited copy).
